## Purpose and Scope

This document describes the multi-language documentation infrastructure for the M5Stack UIFlow MicroPython firmware. The system supports both text-based API documentation (via Sphinx/RST) and visual programming examples (via UIFLOW2 `.m5f2` files), serving developers ranging from beginners using block-based programming to experts using direct MicroPython APIs.

For information about the Python library implementations being documented, see [Hardware Abstraction Libraries](#2). For UIFLOW2 visual programming integration, see [M5UI Library](#3.1).

## Documentation Architecture

The documentation system employs a **parallel dual-language structure** with English (`docs/en/`) and Chinese (`docs/zh_CN/`) documentation trees. Each tree mirrors the codebase's package organization (Unit, Module, HAT, Hardware, M5UI), with master index files organizing content via Sphinx `toctree` directives.

### Reference File Anatomy

A typical `.ref` file contains three categories of substitutions:

**1. Product Image Definitions** - Clickable images linking to product documentation:
```rst
.. |ENV| image:: https://static-cdn.m5stack.com/resource/docs/products/unit/env/env_01.webp
    :target: https://docs.m5stack.com/en/unit/env
    :height: 200px
    :width: 200px
```

**2. UIFLOW2 Block Images** - Screenshots of visual programming blocks:
```rst
.. |init.png| image:: https://static-cdn.m5stack.com/mpy_docs/unit/env/init.png
.. |read_temperature.png| image:: https://static-cdn.m5stack.com/mpy_docs/unit/env/read_temperature.png
```

**3. UIFLOW2 Project Links** - Interactive links opening projects in the UIFLOW2 web editor:
```rst
.. |env_cores3_example.m5f2| raw:: html

    <a href="https://uiflow2.m5stack.com/?example=https://raw.githubusercontent.com/m5stack/uiflow-micropython/develop/examples/unit/env/env_cores3.m5f2" target="_blank">env_cores3_example.m5f2</a>
```

These substitutions are referenced in RST files using the pipe syntax: `|ENV|`, `|init.png|`, `|env_cores3_example.m5f2|`.

## Documentation Content Structure

Each hardware module documentation file follows a standardized template with six sections:

| Section | Content | Code Integration |
|---------|---------|------------------|
| **Title** | Hardware name | `ENV Unit`, `RGB Unit` |
| **Product References** | Images from `.ref` file | `\|ENV\|`, `\|ENV II\|`, `\|ENV III\|` |
| **MicroPython Example** | Literal code block | Direct code listing |
| **UIFLOW2 Example** | Visual block image + link | `\|example.png\|`, `\|env_cores3_example.m5f2\|` |
| **Class Documentation** | API reference | Constructor signatures, method descriptions |
| **UIFLOW2 Blocks** | Per-method block images | `\|init.png\|`, `\|read_temperature.png\|` |

### Code Example Integration

Documentation embeds executable Python examples directly:

```rst
Micropython Example::

    import M5
    from M5 import *
    from unit import *

    M5.begin()

    i2c0 = I2C(0, scl=Pin(1), sda=Pin(2), freq=100000)
    env_0 = ENVUnit(i2c=i2c0, type=1) # ENVUnit

    print(env_0.read_temperature())
    print(env_0.read_humidity())
    print(env_0.read_pressure())
```

The `::` syntax creates a literal code block. While this example is embedded directly, the system also supports `.. literalinclude::` directives to include external files (see next section).

**1. Screenshot Display** - Shows the block-based program:

**2. Interactive Link** - Opens project in UIFLOW2 web IDE (conditionally rendered for HTML output):

The `.. only:: builder_html` directive ensures the interactive link only appears in web documentation, not PDF exports.

### Example File Structure

All example files follow a standardized pattern:

```python
# SPDX headers
import os, sys, io
import M5
from M5 import *
from hardware import *
from unit import ENVUnit

# Global variable declarations
i2c0 = None
env_0 = None

def setup():
    global i2c0, env_0
    
    M5.begin()
    Widgets.fillScreen(0x222222)
    
    i2c0 = I2C(0, scl=Pin(1), sda=Pin(2), freq=100000)
    env_0 = ENVUnit(i2c=i2c0, type=1)
    print(env_0.read_temperature())

def loop():
    global i2c0, env_0
    M5.update()

if __name__ == "__main__":
    try:
        setup()
        while True:
            loop()
    except (Exception, KeyboardInterrupt) as e:
        try:
            from utility import print_error_msg
            print_error_msg(e)
        except ImportError:
            print("please update to latest firmware")
```

This structure separates initialization (`setup()`), main loop logic (`loop()`), and error handling, matching the execution model used by UIFLOW2 code generation.

**Sources: ** [examples/unit/env/env_cores3.py:1-50](https://github.com/m5stack/uiflow-micropython/blob/7af4551a/examples/unit/env/env_cores3.py#L1-L50)

## UIFLOW2 Integration

The documentation system bridges text-based MicroPython programming with UIFLOW2's visual block-based environment through three integration mechanisms:

### Block Image CDN

UIFLOW2 block images are hosted on `static-cdn.m5stack.com/mpy_docs/` and organized by package and class:

```
https://static-cdn.m5stack.com/mpy_docs/unit/env/init.png
https://static-cdn.m5stack.com/mpy_docs/unit/env/read_temperature.png
https://static-cdn.m5stack.com/mpy_docs/unit/rgb/set_brightness.png
```

These images appear in documentation alongside method descriptions, showing the visual equivalent of each Python API call.

**Sources: ** [docs/en/refs/unit.env.ref:14-22](https://github.com/m5stack/uiflow-micropython/blob/7af4551a/docs/en/refs/unit.env.ref#L14-L22), [docs/en/refs/unit.rgb.ref:6-14](https://github.com/m5stack/uiflow-micropython/blob/7af4551a/docs/en/refs/unit.rgb.ref#L6-L14)

```

```

Documentation links these files via the UIFLOW2 web editor URL pattern:

```

```

Clicking this link opens the UIFLOW2 web IDE with the project pre-loaded, enabling users to immediately run or modify visual examples.

### Documentation Validation Strategy

The system ensures documentation accuracy through:

1. **Working Examples** - All example files are executable Python scripts that import from actual implementations
2. **Shared Examples** - Both RST documentation and UIFLOW2 projects reference the same example files
3. **Type Annotations** - Constructor and method signatures include Python type hints (e.g., `i2c: Union[I2C, PAHUBUnit]`)
4. **Consistent Naming** - Class names, method names, and parameter names match exactly between docs and code
