系统采用主从式双核架构，实现上层应用逻辑与底层运动控制的物理分离。
*   **主控核心（Master）：** 采用 M5Stack Fire 主控（基于 ESP32-D0WDQ6-V3）。负责运行高层平衡算法（卡尔曼滤波/互补滤波）、Wi-Fi/蓝牙无线通信、用户交互（屏幕显示、按键）、以及系统任务调度。
*   **运动协处理器（Slave）：** 位于 BALA2 底座内部，采用 STM32F030C8T6（ARM Cortex-M0）。负责实时性要求极高的电机 PID 闭环控制、编码器脉冲计数、舵机 PWM 生成以及电池电压监测。
*   **通信接口：** ESP32 与 STM32 通过 M-BUS 总线上的 I2C 接口进行连接。
    *   **链路定义：** ESP32（SDA: GPIO21, SCL: GPIO22）作为 I2C 主机，STM32 作为 I2C 从机。
    *   **地址设定：** STM32 的 I2C 从机地址配置为 **0x3A**。
    *   **数据流：** ESP32 向 STM32 发送目标速度和转向指令；STM32 向 ESP32 回传当前编码器计数值（里程计）、电池电压状态等遥测数据。

### 2. 电机驱动与反馈电路
底座电路集成了完整的直流电机驱动与闭环反馈系统。
*   **电机驱动芯片：** 使用 **HR8833** 双通道 H 桥电机驱动芯片。
    *   **控制逻辑：** HR8833 的输入端连接至 STM32 的定时器 PWM 输出引脚。通过调节占空比和逻辑电平，控制 A/B 两路通道输出电压的幅度和极性，从而控制电机的转速与正反转。
    *   **保护机制：** 芯片内部集成过热保护和过流保护电路。
*   **N20 减速电机与编码器：**
    *   驱动端连接至 HR8833 的输出端（OUT A1/A2, OUT B1/B2）。
    *   **反馈路径：** 两个 N20 电机均配备 AB 相霍尔正交编码器。编码器的 A 相和 B 相信号线直接连接至 STM32 的硬件定时器输入捕获引脚（Input Capture）。
    *   **闭环控制：** STM32 硬件解码编码器脉冲，计算实时转速与位置，结合 PID 算法（比例-积分-微分）在底层自动调整 PWM 输出，以补偿负载变化和电池电压波动。

### 3. 姿态感知模块 (IMU)
姿态解算是自平衡的核心，依赖于高精度的惯性测量单元。
*   **传感器型号：** **MPU6886**（6 轴 IMU）。
*   **电路连接：** MPU6886 位于 M5Stack Fire 核心内部，挂载于 ESP32 的 I2C 总线（SDA: G21, SCL: G22）上。
*   **数据流向：**
    1.  ESP32 通过 I2C 读取 MPU6886 的原始三轴加速度计（Accelerometer）和三轴陀螺仪（Gyroscope）数据。
    2.  ESP32 内部运行姿态融合算法，计算出机体的实时倾斜角度（Pitch）和角速度。
    3.  解算出的姿态数据作为输入量，通过平衡控制算法计算出保持平衡所需的电机扭矩指令，随后通过 I2C 发送给底座 STM32 执行。

### 4. 电源管理系统
系统采用多级稳压与电源路径管理，确保大功率电机与敏感逻辑电路的稳定运行。
*   **储能单元：** 底座内置 1200mAh 可充电锂聚合物电池。
*   **充电管理：** 包含专用的锂电池充电管理 IC，支持通过 USB 接口输入电源进行恒流/恒压（CC/CV）充电。
*   **电压转换路径：**
    1.  **Boost 升压：** 电池电压（3.7V-4.2V）经过 DC-DC 升压转换器（Boost Converter）提升至 **5V**。该 5V 供电轨连接至 M-BUS 总线，为 Fire 核心和外部 Grove 接口供电，同时也作为伺服舵机的电源。
    2.  **LDO 降压：** 5V 电压经过低压差线性稳压器（LDO）降压至 **3.3V**，为 STM32、ESP32、传感器及闪存芯片供电。
*   **电源共享：** Fire 核心通过 M-BUS 的 5V 和 GND 引脚从底座获取电源。底座电路包含电源开关，用于切断电池到升压电路的路径。

### 5. 舵机驱动扩展
为了支持机械臂或其他扩展结构，底座提供了多路舵机控制能力。
*   **PWM 发生器：** 利用 STM32F030C8T6 的定时器资源生成多路 PWM 信号。
*   **接口布局：** STM32 共引出 8 路舵机信号。
    *   **外部接口：** 4 路标准舵机接口通过物理排针引出，方便连接外部舵机。
    *   **内部预留：** 另有 4 路信号在 PCB 上作为焊盘或内部连接预留。
*   **驱动能力：** 舵机电源直接取自系统 5V 总线（由电池升压而来），信号电平为 3.3V（STM32 GPIO 电平），兼容大多数标准舵机。

### 6. 核心外设与接口 (M5Stack Fire)
Fire 核心板集成了丰富的存储和人机交互外设，通过 ESP32 的 GPIO 矩阵进行连接。

    *   **MicroSD (TF) 卡槽：** 连接至 ESP32 的 SPI 接口，用于大容量数据记录。
*   **显示与音频：**

*   **Grove 扩展接口映射：**
    *   **Port A (红色)：** I2C 总线，引脚为 **G22 (SCL)** 和 **G21 (SDA)**，与内部 IMU 和底座 STM32 共用同一 I2C 物理链路。
    *   **Port B (黑色)：** I/O 扩展接口，引脚为 **G26 (DAC output)** 和 **G36 (ADC input)**，用于模拟量采集或输出。
    *   **Port C (蓝色)：** UART 通讯接口，引脚为 **G16 (RXD)** 和 **G17 (TXD)**，通常用于连接 GPS 或其他串口模块。

### 7. 逻辑电路与通讯
*   **无线通讯电路：** ESP32 芯片内置 RF 电路，外围匹配 3D 天线或板载 PCB 天线，提供 2.4GHz Wi-Fi 和 Bluetooth 4.2/BLE 功能。
*   **USB-TTL 桥接：** Fire 核心板载 CP2104 或 CH9102F 芯片，将 USB 差分信号转换为 UART (TXD0/RXD0) 电平，实现程序下载和串口调试。
*   **电平逻辑：**
    *   系统主要逻辑电平为 3.3V。
    *   ESP32 与 STM32 的 I2C 通信线路上配置了上拉电阻，确保空闲状态电平稳定。
    *   M-BUS 连接器不仅传输电源，还透传了 ESP32 的大部分 GPIO，实现了核心板与底座的电气互联。
