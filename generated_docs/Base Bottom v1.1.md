*   **电池供电开关 (Battery Switch) 逻辑**：

        *   **OFF 状态**：开关断开，电池物理上与 M5-Bus 的 `BAT` 引脚隔离。此设计完全切断了电流路径，防止主机端的电源管理芯片（如 Core1 的 IP5306 或 Core2 的 AXP192）在关机状态下产生漏电流（Leakage Current），从而有效解决了设备长期储存导致电池过放电（Deep Discharge）的问题。
        *   **ON 状态**：开关闭合，电池电压直接加载到 M5-Bus 的 `BAT` 网络上，允许主机从底座电池取电，或通过主机的 USB 输入对底座电池进行充电。
*   **电源轨分布**：
    *   **BAT (Pin 30)**：承载电池电压（3.0V - 4.2V），受物理开关控制。当连接主机时，此网络为主机的 PMIC 提供输入电源或接收充电电流。
    *   **5V (Pin 28)**：VBUS 网络，直接贯通。当主机插入 USB 时，5V 电源通过此引脚传输至底座（虽然 Base Bottom 本身无耗电元件，但可供扩展模块使用）。
    *   **3V3 (Pin 12)**：由主机内部 LDO/DC-DC 转换后的逻辑电压，通过 M5-Bus 引入底座，主要用于向扩展总线上的外设供电。

### 2. M5-Bus 接口与引脚映射

### 4. 关键特性与版本演进

*   **v1.1 版本新增特性 (开关电路)**：
    *   相较于旧版 Core Bottom，v1.1 原理图中最为显著的变化是在电池回路中增加了 **SW (Switch)** 节点。

*   **电气一致性与差异处理**：
    *   **Core1 (Basic/Gray)**：通过 Pin 30 获取电池电压，由内部 IP5306 管理充放电。开关断开时，IP5306 彻底断电。

    *   这种设计保证了底座在电气上对所有 M5Stack Core 系列主机的通用性，无需修改 PCB 即可适配不同的电源管理方案。
