*   **电池充放电管理**：
    *   板载 1.25mm-2P 接口用于连接锂电池。
    *   集成锂电池充电管理 IC，负责从外部电源取电为电池充电，具备恒流/恒压充电控制。
*   **电源维持与软开关逻辑**：
    *   **开机触发**：通过 **WAKE** 按键（物理按下）或 **RTC_INT**（RTC 闹钟中断信号）触发电源通路的开启。
    *   **电源保持 (Latching)**：系统上电后，ESP32-S3 需迅速将 **GPIO46 (HOLD)** 引脚置为**高电平**。该信号控制电源路径上的 MOSFET 或 LDO 使能端，锁定系统供电。
    *   **关机逻辑**：当 **GPIO46** 置为**低电平**时，电源锁存释放，系统断电进入超低功耗状态（电池供电下整机功耗约 6uA），仅保留 RTC 等极低功耗电路维持。
*   **稳压电路**：系统末级采用 LDO 将电压稳压至 **3.3V**，为 MCU、屏幕、传感器等逻辑电路供电。

## 补充信息

基于提供的 **Dial v1.1** 原始文档及硬件特性，针对之前的原理图解析进行以下**技术细节补充**（避免重复已有的引脚映射描述，侧重于电路逻辑与电气特性）：

### 1. 电源锁存与复位关机机制详解
*   **电源保持（Power Latching）原理**：
    *   **启动**：在电池供电模式下，系统处于断电状态。按下 **WAKE** 按键（物理位置位于屏幕模组下方，即文档提到的“屏下按键”）或 **RTC_INT** 信号变为低电平（RTC 定时唤醒），会通过硬件通路导通主电源 MOSFET/LDO 的使能端，系统上电。
    *   **软件锁存**：系统上电后，Bootloader 或应用程序必须迅速将 **GPIO46** 初始化为**输出高电平**。该信号并联在 WAKE 信号的驱动路径上，确保在用户松开按键后，电源使能信号依然保持有效，从而维持系统供电。
    *   **复位即关机机制**：文档中提到的“按下 RST 按键直接关机”在电路原理上是利用了锁存机制的特性。按下 Stamp-S3A 模组上的 **RST** 键会导致 ESP32-S3 芯片硬件复位 -> **GPIO46** 瞬间恢复为默认的高阻抗/输入状态 -> 电源管理电路的使能端失去高电平驱动（配合下拉电阻） -> 电源路径切断 -> 系统掉电。这是一个利用复位信号丢失导致电源锁存失效的硬件设计。

### 2. 电气负载与功耗特性
*   **休眠功耗路径**：文档确认仅电池供电下休眠电流为 **6uA**。在原理图设计上，这表明系统在 GPIO46 拉低后，切断了绝大部分外设（屏幕背光、RFID 射频前端、音频功耗级）的 VCC 供电，仅保留了 **RTC (BM8563)** 和电源管理芯片极微小的静态漏电流路径。
*   **工作电流分布**：USB 供电下约 **88.19mA**，电池供电下约 **100mA**。这主要由 **GC9A01** 屏幕背光和 **ESP32-S3** 的 Wi-Fi 射频收发构成，设计电源电路（Buck/LDO）时需预留至少 500mA 的冗余电流能力以应对 Wi-Fi 突发校准时的峰值电流。

### 3. 版本差异与模组特性 (v1.1 vs v1.0)
*   **主控模组变更**：v1.1 版原理图中的核心组件明确为 **M5Stamp-S3A**。相较于前代，其 PCB 封装层面的主要差异在于**3D 天线**的匹配电路优化以及板载 **Boot 按键**（连接 G0）的机械规格升级（4.0x3.0x2.0mm），提高了通过外壳按压进入下载模式的可靠性。
*   **天线灵敏度**：文档提到的“RFID 标签尺寸小于 Dial 外轮廓可能无法识别”，表明 **WS1850S** 的 13.56MHz 线圈天线设计为了适配圆形 PCB 结构，其耦合系数与天线面积相关，原理图设计中天线匹配电路（电容/电阻网络）是针对标准卡片尺寸调优的。

### 4. 物理交互与电路映射
*   **屏下按键逻辑**：文档中的“屏下按键”在电路图中实际上与 **WAKE** 信号复用。屏幕模组采用浮动结构设计，向下按压屏幕会触发底部的轻触开关，该开关电路上连接至电源唤醒端，实现“按压屏幕唤醒”或“按压屏幕作为输入（需 MCU 上电后检测 WAKE 信号电平）”的功能。
