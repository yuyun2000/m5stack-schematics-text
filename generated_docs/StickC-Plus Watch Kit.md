## 原理图分析

### 1. 核心主控与射频系统
*   **主控芯片**：采用 **ESP32-PICO-D4** 系统级封装 (SiP) 芯片。内部集成了 ESP32 双核 Xtensa® LX6 32 位处理器、晶振、滤波电容以及 4MB SPI Flash。
*   **射频电路**：芯片 RF 引脚连接至 L-C 阻抗匹配网络，随后连接至板载 **2.4G 3D 天线**，支持 Wi-Fi 与蓝牙双模通讯。
*   **时钟系统**：ESP32-PICO 内部已集成 40MHz 晶振，原理图外部电路无需额外的主晶振，仅需RTC电路的时钟源（如果 RTC 芯片独立工作）。

### 2. 电源管理系统 (PMU)
*   **核心 PMU**：采用 **AXP192** 电源管理芯片，负责统筹整机供电。
*   **输入与充电**：
    *   **Type-C 输入**：USB VBUS (5V) 直接输入至 AXP192 作为主电源输入。
    *   **电池管理**：AXP192 的 BAT 引脚连接锂聚合物电池（120mAh 或 200mAh），内置充电控制电路，支持恒流/恒压 (CC/CV) 充电模式。
*   **电源输出分配**：
    *   **DCDC1**：输出 3.3V，为主控 ESP32 及大部分外设供电。
    *   **LDO2**：为 LCD 背光供电，电压可调以实现软件控制背光亮度。
    *   **LDO3**：输出 3.3V，为 LCD 屏幕的逻辑电路供电。
*   **开关机与维持逻辑**：
    *   **开机触发**：**Button C** 连接至 AXP192 的 `N_OE` / `PWRON` 引脚，按下按键拉低电平触发 PMU 启动；同时 **BM8563 (RTC)** 的中断引脚 `INT` 也连接至 AXP192，可实现定时唤醒。
    *   **电源维持 (Power Hold)**：ESP32 的 **GPIO4** 连接至 AXP192 的控制引脚。系统启动后，软件必须将 GPIO4 拉高，向 AXP192 发送“保持供电”信号，否则 PMU 会在超时后自动关机。

### 3. 显示系统
*   **屏幕型号**：1.14 英寸 TFT 液晶屏，驱动 IC 为 **ST7789v2**。
*   **信号接口**：采用 4-wire SPI 接口连接至 ESP32。
    *   **SCK**: GPIO13
    *   **MOSI**: GPIO15
    *   **CS (片选)**: GPIO5
    *   **DC (数据/命令)**: GPIO23
    *   **RST (复位)**: GPIO18
*   **供电设计**：逻辑电压由 AXP192 LDO3 提供，背光阳极由 AXP192 LDO2 提供，通过调整 LDO2 输出电压实现调光。

### 4. I2C 总线与传感器
*   **总线配置**：I2C 总线连接至 ESP32 的 **SDA (GPIO21)** 和 **SCL (GPIO22)**，总线上配置有上拉电阻。
*   **挂载设备**：
    1.  **AXP192**：PMU 本身作为 I2C 从机，ESP32 通过 I2C 读取电池电压、电流数据并配置电源输出。
    2.  **MPU6886**：6 轴惯性测量单元 (IMU)，提供加速度计和陀螺仪数据。
    3.  **BM8563**：实时时钟 (RTC) 芯片，独立于主控计时，并在闹钟触发时通过 `INT` 引脚唤醒 AXP192。

### 5. 音频与交互组件
*   **音频输入 (麦克风)**：
    *   采用 **SPM1423** 数字 PDM (脉冲密度调制) 麦克风。

    *   **DATA**: GPIO34
*   **音频输出 (蜂鸣器)**：
    *   板载无源蜂鸣器连接至 **GPIO2**。电路中包含三极管或 MOSFET 驱动电路以提供足够的驱动电流。
*   **红外发射 (IR)**：
    *   红外发射二极管连接至 **GPIO9**，通过驱动晶体管控制发射。
*   **指示灯**：
    *   红色 LED 连接至 **GPIO10**，通常设计为低电平点亮（Sink 模式）。
*   **按键输入**：
    *   **Button A (M5 键)**：连接至 **GPIO37**。
    *   **Button B (侧键)**：连接至 **GPIO39**。
    *   **Button C (电源键)**：物理连接至 AXP192，用于硬件复位/开关机。
    *   按键电路通常包含 RC 滤波电路以消除抖动。

### 6. 通讯与接口电路
*   **USB 转串口**：
    *   采用 **CH9102** 芯片实现 USB 到 UART 的电平转换。
    *   **数据路径**：USB D+/D- 连接 CH9102，转换后的 TX/RX 连接至 ESP32 的 UART0 (GPIO1/GPIO3)。
    *   **自动下载电路**：利用 CH9102 的 RTS 和 DTR 信号，通过双三极管逻辑电路控制 ESP32 的 EN (复位) 和 IO0 (引导模式) 引脚，实现一键自动下载程序。
*   **GROVE 接口**：
    *   引脚定义：**G32** (SDA/In/Out), **G33** (SCL/In/Out)。

*   **排母扩展接口**：
    *   提供 **G0**, **G25**, **G26** 作为通用 I/O。
    *   提供 **G36** 仅作为输入引脚（ESP32 的 Sensor_VP，仅支持输入/ADC）。
    *   提供 5V, 3.3V, GND 电源轨。

### 7. 逻辑电路特性
*   **上拉/下拉**：
    *   I2C 总线 (GPIO21/22) 设有硬件上拉电阻。
    *   GPIO0, GPIO2 等 Strap 引脚在启动时具备特定的上拉/下拉状态以确立启动模式。
*   **退耦与滤波**：
