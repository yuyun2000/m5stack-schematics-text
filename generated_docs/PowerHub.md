## 原理图分析

以下是 M5Stack PowerHub (SKU:C148/K148) 的原理图技术描述：

### 1. 核心处理与控制架构
系统采用双 MCU 架构，通过分工协作实现高性能计算与精细化电源管理的结合。
*   **主控制器 (Main SoC)**: 采用 **ESP32-S3-WROOM-1U-N16R8** 模组，内置 16MB Flash 和 8MB PSRAM。主要负责 Wi-Fi 无线通信、USB OTG 通讯（连接至引脚 G19/G20）以及上层应用逻辑的处理。
*   **协处理器 (PMU)**: 采用 **STM32G031G8U6** MCU 作为专用电源管理单元 (PMU)。该芯片负责底层硬件电源通断控制、充电逻辑管理以及 USB 信号路径切换。STM32 与 ESP32-S3 之间通过 I2C 总线 (`SYS_SDA`, `SYS_SCL`) 进行通讯交互。

### 2. 电源管理与电压调节
电源系统设计复杂，支持多输入源、双向供电及宽电压调节。
*   **DC-DC 转换**: 核心采用 **SC8721** Buck-Boost（升降压）控制芯片，支持 DC 9V~20V 的宽电压输入与输出。该芯片由 STM32 通过 `PDCDC_EN` (使能) 和 `PDCDC_REFLOW` (回流控制) 信号进行管理，以适应不同的电源拓扑。
*   **电源路径**: 系统支持三种供电方式：DC Jack (5.5x2.1mm) 接口输入、2S 锂电池 (兼容 NP-F550/750/950 系列) 供电，以及通过 RS485/CAN 接口实现的双向供电（既可由本机向总线供电，也可由总线向本机供电）。
*   **电源分配控制**: 各路电源输出由专用 GPIO 控制的开关电路管理：
    *   `OEN_USB`: 控制 USB 端口供电。
    *   `OEN_GRV_R`: 控制 Port A (Grove) 供电。
    *   `OEN_GRV_B`: 控制 Port C (Grove) 供电。
    *   `OEN_PWROUT`: 控制向 RS485/CAN 接口输出电源（Forward Power）。
    *   `OEN_5VO`: 控制 2.54mm 16P 扩展总线上的 5V 输出。

### 3. 电源监测系统
系统集成了高精度的电源遥测电路，用于实时监控各路电压与电流。
*   **监测芯片**: 采用 6 颗 **INA226** 电流/电压监测 IC，挂载于专用的 I2C 总线 (`PM_SDA`, `PM_SCL`) 上，地址分配为 0x40 至 0x46。
*   **监测节点**: 分别监测 USB 接口、Port A、Port C、PWRCAN 总线电源、PWR485 总线电源以及电池 (Battery) 回路的电压与电流状态。
*   **输入检测**: STM32 通过 `VIN_DET` 信号直接检测外部 DC 输入状态。

### 4. 电池充电与 RTC
*   **充电管理**: STM32 控制充电电路，通过 `CHG_EN` 信号使能充电，并通过 `CHG_STAT` 读取充电状态。电路支持通过 USB-C 输入对 2S 锂电池进行涓流和恒流/恒压充电。
*   **实时时钟 (RTC)**: 板载 **RX8130CE** RTC 芯片，并配备 70000μF 超级电容作为掉电后备电源。RTC 具备低功耗唤醒功能，通过 `nSTBY_WAKE_UP` 信号唤醒主控系统。

### 5. 通讯接口与信号链路
*   **USB 矩阵切换**: 设计了二级 USB 信号开关矩阵，由 STM32 的 `USB_CON_LV1` 和 `USB_CON_LV2` 信号控制。该矩阵负责将 ESP32-S3 的 USB D+/D- 信号动态路由至以下三个物理端口之一：底部 Type-C 输入/OTG 接口、前置 Type-C 输出接口或前置 Type-A 输出接口。
*   **工业总线**:
    *   **RS485**: 集成 RS485 收发器，连接至 ESP32 的 G8 (TX), G17 (RX), G18 (RTS/EN)。
    *   **CAN**: 集成 CAN 收发器，连接至 ESP32 的 G39 (TX), G40 (RX)。
    *   **终端匹配**: RS485 和 CAN 接口均配备物理拨动开关，用于接入或断开 120Ω 终端电阻。

### 6. 外设与扩展接口
*   **人机交互**:
    *   8 颗 **WS2812** RGB LED 串联，由 STM32 通过 `LED_DATA` 引脚驱动。
    *   物理按键包括电源键 (`BtnPWR`，长按可进入下载模式)、PMU 功能键 (`PMU_SW1`, `PMU_SW2`) 以及用户自定义按键 (`USR_SW2`)。
*   **扩展连接**:
    *   **Grove 接口**: 提供 2 个 HY2.0-4P 接口（Port A 和 Port C），支持 M5Stack 标准外设。
    *   **EXT Bus**: 2.54mm 间距的 16Pin 排母接口，引出了关键电源轨（BAT-2S, HVIN, 5VOUT）及多路 ESP32 GPIO，便于用户进行二次开发或连接扩展模块。

    *   **Port C (蓝色)**: UART/IO 信号连接至 ESP32 的 **G1** (TX) 和 **G2** (RX)。
*   **用户交互硬件映射**:
    *   **用户按键 (User Button)**: 黄色圆形按键连接至 ESP32 的 **G11** (`USR_SW2`)。
    *   **PMU 按键**: STM32 直接读取两个按键：侧面按键 (`PMU_SW2`) 连接 **PA4**，顶部半透明按键 (`PMU_SW1`) 连接 **PA2**。
    *   **RGB LED 控制**: 除了数据引脚 **PA7** (`LED_DATA`)，STM32 还通过 **PB4** (`LED_EN`) 控制 LED 的总电源使能。

### 2. 电源控制逻辑与状态机细节
原理图包含具体的控制逻辑电平定义，用于管理双向能量流动：
*   **控制引脚定义**:
    *   `nVA_EN` (PA1): INA226 监测电路电源/使能控制。
    *   `OEN_USB` (PB8), `OEN_GRV_R` (PC14), `OEN_GRV_B` (PC15): 分别控制 USB、Port A、Port C 的电源输出。
*   **双向供电逻辑**:
    *   **正向输出模式 (Forward Power)**: STM32 将 `PDCDC_EN` (PC6) 置高（开启 DCDC），`OEN_PWROUT` (PB1) 置高（导通输出），同时必须将 `PDCDC_REFLOW` (PA8) 保持为 **低电平**。此模式下需开启 `nVA_EN`。
    *   **反向输入模式 (Reverse Power/Reflow)**: STM32 将 `PDCDC_EN` 置低（关闭 DCDC），`OEN_PWROUT` 置低（断开输出），并将 `PDCDC_REFLOW` (PA8) 置为 **高电平**。此时允许外部电源通过 RS485/CAN 接口倒灌给主机供电。

### 3. 连接器物理规格与总线引脚定义
*   **物理接口规格**:
    *   **RS485 接口**: 采用 **VH3.96-4P** 连接器。
    *   **CAN 接口**: 采用 **XT30 (2+2)** 大电流连接器。
    *   **无线接口**: 板载 **SMA** (内螺内孔) 天线接口，用于连接 2.4GHz Wi-Fi 天线。
*   **EXT 2.54-16P 总线完整定义**:
    *   **电源**: Pin 1/2 (BAT-2S), Pin 3/4 (HVIN, 高压输入), Pin 5/6 (GND), Pin 4 (5VOUT)。
    *   **控制**: Pin 8 (RST), Pin 10 (nWKUP)。
    *   **GPIO**: Pin 7 (G43), Pin 9 (G44), Pin 11 (G42), Pin 13 (G41), Pin 12 (G7), Pin 14 (G6), Pin 15 (G4), Pin 16 (G5)。

### 4. USB 矩阵开关控制细节
USB 切换矩阵由 STM32 的两个 GPIO 精确控制：
*   **USB_CON_LV1 (PB3)**: 负责一级切换，决定 ESP32 USB 信号是连接到底部 Type-C (输入/OTG) 还是进入二级开关。
*   **USB_CON_LV2 (PA15)**: 负责二级切换，将信号路由至前置 Type-C 输出或前置 Type-A 输出。

### 5. 唤醒机制补充
*   **RTC 唤醒**: RX8130CE RTC 芯片的中断输出连接至 STM32 的 **PB5** (`nSTBY_WAKE_UP`)，用于实现低功耗定时唤醒功能。
