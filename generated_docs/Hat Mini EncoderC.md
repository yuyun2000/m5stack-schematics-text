*   **通信角色**：MCU 被配置为 **I2C 从机**设备，默认 I2C 地址为 **0x42**。通过 I2C 总线接收来自 StickC/StickC-Plus 主机的配置指令，并上报编码器计数值、按键状态及电池电压数据。

    *   **按键输入**：PA2 引脚连接编码器集成的按键开关。
    *   **LED 控制**：PA5 引脚作为输出，驱动 SK6812 RGB LED 的数据输入端 (DIN)。
    *   **电池监测**：PA4 (ADC_IN4) 连接电池电压分压电路，用于模拟量采样。
    *   **调试接口**：PA13 (SWDIO) 和 PA14 (SWCLK) 引脚引出至测试点，用于固件烧录与调试。

### 2. 旋转编码器与按键电路
*   **编码器结构**：使用 30 位脉冲的机械式旋转编码器。
*   **信号处理**：编码器的 A 相和 B 相引脚直接接入 MCU 的 GPIO（PA0, PA1）。电路设计依赖 MCU 内部的上拉电阻或外部上拉电阻将空闲电平拉高。MCU 通过检测 A/B 相位的电平跳变顺序（相位差）来判断旋转方向（顺时针/逆时针），并通过脉冲计数计算旋转数值。
*   **按键逻辑**：编码器中心轴集成的按键开关一端接地，另一端连接 MCU 的 PA2。当按键按下时，PA2 电平被拉低（Active Low）。MCU 内部通过消抖逻辑处理该信号以触发按键事件。

### 3. 电源与电池管理系统
*   **电源输入**：主电源输入来自 StickC 顶部的 8 Pin 接口或 HY2.0-4P 接口的 **5V** 总线。
*   **充电管理**：电路包含一颗线性锂电池充电管理 IC（通常为 ME4054 或同类 SOT-23-5 封装芯片）。5V 输入经过该 IC 对内置的 200mAh 聚合物锂电池进行恒流/恒压（CC/CV）充电。
*   **系统供电与稳压**：
    *   **电源开关**：电池正极路径上串联了一个单刀双掷（SPDT）滑动开关，用于物理切断电池对系统的供电。
    *   **电压转换**：电池电压（VBAT）经过一颗低压差线性稳压器（LDO，3.3V 输出）转换为 **3.3V**，为 STM32F030F4P6 及相关逻辑电路供电。
*   **电压监测**：电池正极（VBAT）通过电阻分压网络连接至 STM32 的 PA4 引脚。MCU 通过 ADC 读取分压后的电压值，换算后通过 I2C 寄存器向主机提供当前电池电压信息。

### 4. 通讯接口与协议
*   **物理接口**：产品具有两组并联的接口：

    2.  **HY2.0-4P (Grove) 接口**：用于通过线缆连接外部主机。
*   **信号定义**：接口中的 5V、GND、SDA、SCL 网络全部并联。
    *   **SDA**：连接至 STM32 的 PA10。
    *   **SCL**：连接至 STM32 的 PA9。
*   **总线配置**：I2C 总线（SDA 和 SCL）上配置了上拉电阻（通常为 4.7kΩ 或 10kΩ），以确保空闲状态下的高电平，保证通信稳定性。

### 5. 可编程 RGB LED 电路
*   **元件选型**：板载一颗 **SK6812-3535** 封装的可编程 RGB LED。
*   **驱动方式**：LED 采用单线串行通信协议。STM32 的 PA5 引脚输出控制信号至 LED 的 DIN 引脚。
*   **供电**：LED 的 VDD 引脚连接至 5V 电源轨（通过 StickC 接口供电）或在部分设计中通过电池升压/直连供电，确保足够的驱动电压。

### 6. 逻辑电平与保护

*   **接口保护**：I2C 通信线直接连接至 MCU 引脚，主要依赖 MCU 内部的保护二极管进行基本的 ESD 防护。
*   **电源域划分**：
    *   **5V 域**：用于电池充电输入和 RGB LED 供电（视具体版本而定）。
    *   **VBAT 域**：电池直接输出，经过开关。

### 2. I2C 协处理架构与时序卸载
*   **硬件抽象层**：在电路架构上，STM32 充当了 **I2C 智能外设控制器**（协处理器）。
    *   **实时性解耦**：编码器的正交脉冲计数（需高频中断处理）和 SK6812 LED 的驱动（需严格微秒级时序）完全由 STM32 本地硬件定时器及固件处理。
    *   **寄存器映射**：STM32 将这些实时任务抽象为静态的 I2C 寄存器（如文档所示的 0x42 地址下的数据），主机（StickC）仅需通过标准的 I2C 读写操作即可获取计数值或更新 LED 颜色，无需占用主机 CPU 资源进行底层信号解码。

### 3. 充电电流编程 (PROG)
*   **电流设定**：在锂电池充电管理电路（ME4054 等效部分）中，**PROG** 引脚连接有一颗高精度**编程电阻**接地。该电阻的阻值经过计算，将恒流充电电流（ICC）限制在适合 200mAh 电池的倍率范围内（通常设定在 100mA - 200mA 之间），以确保充电安全并延长电池寿命。

### 4. 逻辑电平匹配
*   **直接耦合设计**：由于 StickC 主机的 ESP32 I/O 电平和 STM32F030 的供电电压均为 **3.3V**，原理图在 I2C 总线（SDA/SCL）连接上采用**直接耦合**方式，未设置 MOSFET 电平转换电路。这要求外部通过 HY2.0-4P 接口接入 5V 主机（如某些 Arduino Uno）时，必须确保外部主机的 I2C 总线电平已转换为 3.3V，否则可能损坏 MCU。
