### 2. M5-Bus 接口与 SPI 通信逻辑
该模块通过标准的 M5-Bus 2x15 Pin 排母与 M5Stack 主机（Core）连接，主要采用 SPI 协议进行通信与控制。具体管脚映射如下：

    *   **SCK/CLK (Serial Clock)**：连接至 M5-Bus 的 **Pin 11** (对应 ESP32 GPIO 18)，由主机提供 SPI 通信时钟信号，决定通信速率（最高支持 10MHz，进而支持高达 300Kbps 的可编程比特率）。
*   **控制与中断信号**：
    *   **CS/NSS (Chip Select)**：连接至 M5-Bus 的 **Pin 20** (对应 ESP32 GPIO 5)。低电平有效，用于选中 Ra-02 模组进行 SPI 通信。
    *   **RST (Reset)**：连接至 M5-Bus 的 **Pin 8** (对应 ESP32 GPIO 26)。主机可通过拉低该引脚复位 SX1278 芯片，用于初始化或错误恢复。

*   **滤波与去耦**：
    *   在 LDO 的输入端和输出端分别布置了滤波电容（通常为 MLCC），用于滤除电源噪声，防止高频纹波干扰射频信号。

### 5. 逻辑电路与物理结构
*   **逻辑电平**：由于 M5Stack Core 主控（ESP32）和 Ra-02 模组均为 3.3V 逻辑电平，原理图中 SPI 总线及控制信号线（CS, RST, IRQ）采用直连方式，未设计额外的电平转换电路。
