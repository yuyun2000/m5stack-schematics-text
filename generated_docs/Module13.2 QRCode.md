## 原理图分析

**主要芯片与核心组件**
*   **扫描模组**：核心采用 640x480 CMOS 成像扫描引擎，内部集成白色照明 LED、红色瞄准 LED 以及蜂鸣器。支持二维码与条形码识别。
*   **IO 扩展芯片 (PI4IOE5V6408)**：通过 I2C 总线（地址 0x44）与主机通讯。主要用于扩展 GPIO 控制信号：
    *   **P0 引脚**：输出 `QR_5V_EN` 信号，用于控制扫描模组的电源使能。
    *   **P4 引脚**：输出 `TRIG` 信号，连接至扫描模组的硬件触发引脚，用于启动扫描任务。

**电源管理与供电电路**
*   **宽电压输入**：板载 5.5x2.1mm DC 接口，支持 DC 9V ~ 24V 的宽电压输入。
*   **降压电路 (DC-DC)**：内部包含 DC-DC 降压转换电路，将外部输入的 9-24V 电压转换为系统所需的 5V 电源。
*   **电源分配**：降压后的 5V 电源通过电源开关逻辑连接至 M5-Bus 的 HPWR 和 5V 电源轨，为扫描模组供电的同时，也可通过 M5-Bus 为堆叠的主机供电。

**扫描引擎控制逻辑**
*   **触发机制**：扫描触发信号 `TRIG` 由 IO 扩展芯片控制，支持手动脉冲触发、自动感应触发及连续扫描模式（取决于主机下发的 I2C 指令及模组配置）。
*   **反馈系统**：扫描模组内部集成的蜂鸣器与 LED 直接由模组内部逻辑驱动，作为扫描成功或对准的物理反馈。
*   **控制总线**：主控通过 M5-Bus 的 GPIO 17 (SDA) 和 GPIO 18 (SCL) 访问 PI4IOE5V6408 扩展芯片，实现对模组供电和触发的控制。

**通讯接口与信号切换电路**
*   **多协议支持**：扫描模组数据接口支持 UART 和 USB 两种物理层协议。USB 模式下支持虚拟串口、HID 及 HID-POS 功能。
*   **物理信号切换矩阵**：电路板上设计有多个物理拨动开关（SW），用于切换 `QR_TX`/`QR_RX`（UART）与 `USB_DP`/`USB_DM`（USB）的信号路径。
*   **M5-Bus 信号映射**：为适应不同 M5Stack 主机的引脚占用情况，通过开关可以将 UART 数据信号（TX/RX）路由至 M5-Bus 的不同 GPIO 组，包括：
    *   GPIO 13 / 14
    *   GPIO 15 / 16
    *   GPIO 21 / 22
    *   GPIO 23 / 26
*   **PORT.C 接口复用**：外部 PORT.C 接口根据开关配置，可被定义为 UART 通讯接口（TX/RX）或 USB 通讯接口（DP+/DM-），用于连接电脑或其他 USB 主机。

**外部接口与扩展 (Grove Ports)**
*   **PORT.A (红色)**：专用 I2C 接口，信号线定义为 SDA 和 SCL，与内部 I2C 总线及 IO 扩展芯片并联，可用于外接传感器。
*   **PORT.B (黑色)**：通用 I/O 接口，提供通用输入/输出功能。
*   **PORT.C (蓝色)**：通讯专用接口，如上所述，支持 UART 或 USB 信号的物理切换与输出。

**电路连接关系总结**
*   **控制路径**：M5-Bus (I2C) -> PI4IOE5V6408 -> 扫描模组 (TRIG/EN)。
*   **数据路径**：扫描模组 (TX/RX/DP/DM) -> 拨动开关矩阵 -> M5-Bus GPIO 或 PORT.C。
*   **供电路径**：DC Jack -> DC-DC Buck -> 5V Rail -> M5-Bus HPWR & 扫描模组电源输入 (受控于 EN)。

**信号路由与引脚映射细节**
*   **PORT.B 连接**：根据管脚映射表，Grove 接口 PORT.B 的信号线（IN/OUT）直接路由至 M5-Bus 的 **Pin 4** 和 **Pin 10**。
*   **PORT.C 信号复用**：PORT.C 接口的 Yellow 和 White 引脚具备物理层协议切换功能：
    *   **UART 模式**：Yellow 定义为 TX，White 定义为 RX。
    *   **USB 模式**：Yellow 复用为 USB_DP+，White 复用为 USB_DM-，允许通过 Grove 线缆转接 USB-C 连接上位机。
*   **开关矩阵逻辑**：板载拨动开关（SW）具备**断开（NC）**功能。当开关拨至特定档位时，对应的 M5-Bus 引脚（如 GPIO 13/14, 21/22, 25/26 等）与扫描模组断开连接，此设计用于在堆叠多模块时避免引脚冲突。

**扫描模组光学参数**
*   **识读性能**：扫描引擎支持 **50 ~ 160 mm** 的景深范围，识别精度 **≥ 5mil**。
*   **扫描角度**：光学组件支持 **±55°** 的仰角与偏角扫描。
*   **码制支持**：硬件解码逻辑覆盖 PDF417、QR Code、Data Matrix 等二维条码，以及 Code128、EAN-13、UPC-A 等主流一维条码。
