*   **电源输入**：通过 14-pin **Cap-Bus** 接口的 5V_IN 引脚从主控设备获取电源。
*   **电压转换**：电路板上包含 **LDO (低压差线性稳压器)** 电路，将输入的 5V 电压转换为 **3.3V** 系统电压，分别为 SX1262 LoRa 模组和 AT6668 GNSS 芯片供电。
*   **电源滤波**：在 3.3V 电源轨及模组电源输入端配置有退耦电容，以滤除高频噪声并满足 LoRa 发射瞬间及 GPS 搜星时的高电流需求（峰值约 155mA），保证电源完整性。

*   **GNSS UART 通讯接口**：
    *   AT6668 通过异步串口与主控交互 NMEA 数据：模组的 TXD 连接至主控的 **G15** (RX)，模组的 RXD 连接至主控的 **G13** (TX)。通讯波特率设定为 115200bps。
*   **Cap-Bus 总线接口**：
    *   采用 2x7 (14-pin) 排针/排母连接器，除上述 SPI 和 UART 信号外，还包含电源引脚及预留的 I2C 总线引脚，用于物理连接 Cardputer-Adv 主机。

### 4. 射频前端与天线电路
*   **LoRa 射频链路**：SX1262 的射频输出/输入引脚经过阻抗匹配网络，连接至板载的 **SMA 天线连接器**。该路径设计针对 868MHz 频段进行了优化，需配合外置 SMA 天线使用以保证通信距离。
*   **GNSS 射频链路**：AT6668 的射频输入端直接连接至板载 **陶瓷天线**。电路设计包含必要的无源匹配元件，以优化卫星信号的接收灵敏度。

### 5. 逻辑电平与兼容性
*   **电平标准**：系统逻辑电平设计为 **3.3V**。由于主控设备 (ESP32-S3) I/O 同样为 3.3V，SPI 和 UART 总线采用直连方式，无需额外的逻辑电平转换芯片。
*   **信号隔离**：LoRa 的 SPI 总线与 GNSS 的 UART 总线在物理引脚上独立分配，互不干扰，支持两个模组同时工作。
