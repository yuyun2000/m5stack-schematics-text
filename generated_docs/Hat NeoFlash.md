## 原理图分析

以下是基于 M5Stack Hat NeoFlash (SKU: U071) 硬件原理图的详细技术描述：

### 1. 核心显示电路与阵列结构
*   **核心元件**：电路包含 126 颗 WS2812B（或兼容型号）智能控制 LED 光源，物理排列为 7行 x 18列 的矩阵布局。
*   **级联拓扑（Daisy-Chain）**：所有 LED 采用单总线串行级联方式连接。
    *   **信号入口**：来自接口端的控制信号直接连接至矩阵中第 1 颗 LED（Pixel 0）的 **DIN**（Data Input）引脚。
    *   **级联路径**：第 1 颗 LED 的 **DOUT**（Data Output）引脚连接至第 2 颗 LED 的 DIN 引脚，依此类推，直到第 126 颗 LED。
    *   **信号整形**：每一级 LED 内部包含信号整形放大电路，接收到的信号经过整形后再通过 DOUT 传递给下一级，防止信号随级联深度增加而衰减，确保 126 个节点的信号完整性。

### 2. 管脚映射与接口定义
*   **接口形式**：采用标准 8-Pin 排母接口（Hat Header），用于直接连接 M5StickC 控制器。
*   **信号定义**：
    *   **G26 (GPIO 26)**：定义为 **DATA** 信号线。这是唯一的逻辑控制引脚，承载 NZR（归零码）通讯协议信号，负责驱动整个 LED 阵列的数据传输。
    *   **5V**：电源输入引脚，直接连接至板载 5V 电源总线。
    *   **GND**：公共地线，连接至所有 LED 的 VSS 引脚和去耦电容的接地端。
    *   **NC (Not Connected)**：接口中的 3V3、G0、G36 等引脚在 LED 驱动逻辑中未进行电气连接（或仅作直通处理），不参与显示控制。

### 3. 电源管理与去耦系统
*   **供电架构**：系统由 M5StickC 通过 5V 引脚提供直流电源。电源以并联总线方式分布至 126 颗 LED 的 VDD 引脚。
*   **去耦设计**：
    *   原理图中在 LED 阵列的电源输入端及局部区域配置了去耦电容（通常为 100nF / 0.1µF 陶瓷电容）。
    *   **功能**：用于滤除 WS2812 内部 PWM 恒流驱动器在高速开关切换时产生的电源噪声，并为 LED 瞬间启动提供瞬态电流补偿，防止电压跌落导致显示颜色异常或逻辑混乱。

### 4. 控制逻辑与通信机制
*   **通信协议**：基于单线归零码协议。
*   **数据传输**：
    *   控制器（M5StickC）通过 G26 发送 24-bit 数据流。
    *   每个 LED 截取首个 24 位数据（G: 8-bit, R: 8-bit, B: 8-bit）并存入内部锁存器，剩余数据通过 DOUT 转发至下一个级联像素。
*   **色彩与亮度**：
    *   硬件支持每种基色（红、绿、蓝）256 级（8位）亮度调节。
    *   通过三基色混合，实现 16,777,216 种真彩色（True Color）显示。
    *   数据刷新频率通常为 400Hz 或 800Hz，取决于具体使用的 WS2812 版本。

### 5. 电路连接关系总结
*   **输入级**：Hat Header Pin (G26) -> Resistor (可选限流电阻) -> LED_1 (DIN)。
*   **中间级**：LED_n (DOUT) -> LED_n+1 (DIN)。
*   **电源级**：Hat Header Pin (5V) -> Power Bus -> LED_All (VDD)；Hat Header Pin (GND) -> Ground Plane -> LED_All (VSS)。
*   **物理特性**：电路设计适应 58x23.5mm 的 PCB 尺寸，排针设计为 90° 弯曲或直插（视具体版本），使得扩展板能紧贴 M5StickC 屏幕背部或顶部，形成紧凑的显示扩展模块。
