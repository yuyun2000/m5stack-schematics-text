*   **核心元件**: 包含一个机械轴体（通常为 Blue Switch 青轴），作为物理输入设备。
*   **电路拓扑**: 采用**低电平触发 (Active Low)** 的“对地开关”结构。
    *   按键的一端直接连接至接口的 White 引脚。
    *   按键的另一端直接连接至 GND。
*   **工作逻辑**:
    *   **按键释放**: 物理触点断开，White 引脚处于高阻/悬空状态。此时电路依赖主控端（MCU）GPIO 的内部上拉电阻将电平拉高至逻辑高电平。
    *   **按键按下**: 物理触点闭合，White 引脚被直接短路至 GND，输出逻辑低电平 (0V)。

### 3. RGB LED 控制电路
*   **核心元件**: 内置一颗 SK6812 可编程全彩 RGB LED。
*   **供电设计**: LED 的 VDD 引脚直接连接至接口的 5V 电源线，确保 LED 驱动电路获得足够的驱动电压。
*   **信号路径**: 接口的 Yellow 引脚连接至 SK6812 的 DIN (Data Input) 引脚。LED 内部集成控制芯片，通过单线串行通讯协议接收颜色数据。
*   **电源保护**: 在 SK6812 的 VDD 与 GND 引脚附近并联有一颗去耦电容（通常为 0.1uF），用于滤除电源线上的高频噪声，防止 LED 在 PWM 调光或颜色切换瞬间引起的电压波动影响稳定性。

### 4. 电平与电源转换分析
*   **按键逻辑电平 (3.3V 兼容性)**: 原理图中并未采用电阻分压来实现 3.3V 转换，而是采用了**开漏/开集 (Open Drain)** 的电路设计思想。由于按键闭合时仅将信号线拉低至 GND，而不闭合时信号线悬空，因此输出的高电平电压完全由主控端 GPIO 的上拉参考电压决定。当主控端为 ESP32 等 3.3V 系统时，上拉即为 3.3V，从而在 5V 供电的条件下，安全地向主控输出符合 3.3V 标准的逻辑信号。
*   **LED 驱动电平**: 虽然 LED 由 5V 供电，但 SK6812 的数据输入端 (DIN) 通常能够识别 3.3V 的高电平逻辑信号，因此可直接由主控驱动。

### 5. 整体逻辑架构
Unit Key 的硬件架构由两个相对独立的回路组成，共用电源地：
1.  **输入链路**: 物理按压 -> 机械触点闭合 -> White 引脚对地导通 -> 主控检测到下降沿/低电平。
2.  **输出链路**: 主控发出数据流 -> Yellow 引脚传输 -> SK6812 解码 -> LED 发光显示。
该设计通过最简化的电路实现了单按键输入与可编程灯光反馈的集成，并将所有逻辑电平处理交由主控端配置，属于典型的无源输入加有源外设的组合设计。
