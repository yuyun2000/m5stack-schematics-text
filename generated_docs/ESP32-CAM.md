*   **主控芯片**：采用 ESP32-D0WDQ6（或同系列）双核处理器，集成 Wi-Fi 和蓝牙功能。
*   **存储配置**：
    *   **Flash**：配置 4MB SPI Flash，用于存储程序固件及文件系统。
    *   **SRAM**：芯片内部集成 520KB SRAM，作为系统运行内存。由于未外挂 PSRAM，图像处理缓冲区受限于内部 SRAM 大小，限制了高分辨率视频流的帧率。
*   **晶振电路**：ESP32 连接外部 40MHz 晶体振荡器，为系统提供主时钟源。

### 2. 摄像头子系统 (OV2640)
该部分电路通过 DVP (Digital Video Port) 接口与 ESP32 直接相连，包含数据传输、时钟控制及配置总线。
*   **图像传感器**：OV2640，200万像素 CMOS 传感器。
*   **数据总线 (DVP 8-bit)**：
    *   OV2640 的并行像素数据引脚 D0-D7 映射至 ESP32 的 GPIO，具体连接如下：
        *   D0 -> GPIO17
        *   D1 -> GPIO35
        *   D2 -> GPIO34
        *   D3 -> GPIO5
        *   D4 -> GPIO39
        *   D5 -> GPIO18
        *   D6 -> GPIO36
        *   D7 -> GPIO19
*   **同步与时钟信号**：
    *   **XCLK (System Clock)**：由 ESP32 的 GPIO27 输出高频时钟信号驱动摄像头工作。
    *   **PCLK (Pixel Clock)**：摄像头输出的像素时钟，连接至 GPIO21。
    *   **VSYNC (Vertical Sync)**：帧同步信号，连接至 GPIO22，用于标识新的一帧图像开始。
    *   **HREF (Horizontal Reference)**：行参考信号，连接至 GPIO26，用于标识一行有效数据。
*   **配置总线 (SCCB)**：
    *   采用类 I2C 协议对 OV2640 寄存器进行配置。
    *   SIOC (Clock) -> GPIO23
    *   SIOD (Data) -> GPIO25
*   **电源与复位控制**：
    *   **RESET**：连接至 GPIO15，用于软件复位摄像头模块。
    *   **PWDN (Power Down)**：OV2640 的 Pin 8 (PWDN) 通过 **12KΩ 下拉电阻** 接地。电路设计上强制摄像头处于使能（Enable）状态，默认工作，不通过 GPIO 进行动态电源开关控制。

### 3. USB 通讯与自动下载电路
*   **USB-TTL 桥接**：采用 Silicon Labs **CP2104** 芯片。
    *   将 USB 接口的差分信号（D+, D-）转换为 TTL 电平的 UART 信号。
    *   TXD 连接至 ESP32 的 U0RXD (GPIO3)。
    *   RXD 连接至 ESP32 的 U0TXD (GPIO1)。
*   **自动下载电路**：
    *   利用 CP2104 的 DTR 和 RTS 信号，通过由两个三极管构成的逻辑电路控制 ESP32 的 **EN** (复位) 和 **IO0** (启动模式) 引脚，实现由 IDE 触发的一键自动烧录程序功能，无需手动按键。

### 4. 接口与扩展电路
*   **HY2.0-4P 接口 (Grove)**：
    *   **电源**：提供 5V 电源输入/输出及 GND。
    *   **通讯**：I2C 总线，定义为 **SCL (GPIO4)** 和 **SDA (GPIO13)**，用于连接 M5Stack Unit 系列传感器或其他 I2C 设备。
*   **预留传感器焊盘**：
    *   原理图上预留了环境传感器 **BME280** (气压、温湿度) 和惯性测量单元 **MPU6050** (6轴 IMU) 的封装位置，它们共用上述 I2C 总线。
    *   **麦克风**：预留 MEMS 麦克风 **SPQ2410** 电路，其模拟输出信号连接至 **GPIO32**，利用 ESP32 的 ADC 功能进行音频采样。

### 5. 电源管理
*   **输入电源**：
    *   主电源来自 USB Type-C 接口 (5V VBUS) 或 HY2.0-4P 接口的 5V 引脚。
*   **电压转换 (LDO)**：
    *   使用低压差线性稳压器 (LDO) 将输入的 5V 转换为 **3.3V**。
    *   3.3V 电源轨为 ESP32 主控、OV2640 摄像头 (AVDD/DVDD)、CP2104 I/O 及 Flash 供电。
*   **滤波**：在 LDO 的输入端和输出端，以及各芯片的电源引脚处，配置了不同容值的陶瓷电容（如 10uF, 100nF）进行滤波和去耦，以保证电源稳定性，降低摄像头图像噪点。

### 6. 指示与用户交互
*   **LED 指示灯**：连接至 **GPIO16**。通过控制 GPIO16 的电平高低来控制 LED 的亮灭，通常用于状态指示或调试反馈。

### 7. 信号完整性与设计逻辑
*   由于视频流数据量大，ESP32 采用 DMA 方式通过 GPIO 直接采集 DVP 数据。原理图中 XCLK (GPIO27) 走线通常较短且包地处理以减少干扰。
*   由于没有外部 PSRAM，图像缓存完全依赖内部 RAM，因此电路设计上仅支持较低分辨率或较低帧率（5-6 FPS）的视频流传输，以避免内存溢出和总线拥塞。
