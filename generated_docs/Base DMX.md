## 原理图分析

### 1. 电源管理与电压调节系统
*   **输入电源**：设备通过 DC-JACK 接口接收外部电源输入，标号为 `HPWR`，支持宽电压输入范围（DC 9V ~ 24V）。该电源轨直接通过 M5-Bus 上的 HPWR 引脚与 M5 主机相连。
*   **DC-DC 降压电路**：采用 **MP1584EN** 高频降压开关稳压器，将输入的 HPWR 高电压转换为系统所需的 `5V` 主电源。该电路包含必要的电感、电容滤波网络及反馈电阻，以维持电压稳定。
*   **电源隔离设计**：为了实现 DMX 总线与逻辑控制侧的电气隔离，电路使用了 **B0505LS-1WR2** 隔离型 DC-DC 电源模块。该模块的输入端连接系统主 `5V`，输出端产生独立的悬浮电源 `VCC_5V_ISO` 和隔离地 `ISO_GND`。此隔离电源专门为 RS-485 收发器芯片及光耦的次级侧供电，彻底切断了地环路干扰路径。

### 2. 核心通讯架构与电平转换
*   **RS-485 收发器**：核心通讯采用 **SP485EEN** (或 SP3488EN) 芯片，负责将 TTL 逻辑电平转换为 RS-485 差分信号（A/B 线）。该芯片由隔离电源 `VCC_5V_ISO` 供电。
*   **收发控制**：收发器的使能引脚（RE/DE）受控于 M5 主机的 **GPIO 21** (`RS485_EN`)。
    *   当 GPIO 21 置高时，驱动器使能，允许数据发送。
    *   当 GPIO 21 置低时，接收器使能，允许数据接收。

### 3. 信号隔离设计
为了保护 M5 主机并确保工业级通讯的可靠性，M5-Bus 与 RS-485 总线之间通过光电耦合器进行信号隔离：
*   **发送路径 (TX)**：M5 主机的 **GPIO 2** (`UART_TX`) 信号连接至高速光耦 **EL0600** 的阳极。光耦输出端驱动 SP485 芯片的 DI (Driver Input) 引脚。该路径实现了从逻辑侧到总线侧的单向隔离传输。
*   **接收路径 (RX)**：SP485 芯片的 RO (Receiver Output) 引脚连接至光耦 **EL0631** 的输入侧。光耦输出端连接至 M5 主机的 **GPIO 22** (`UART_RX`)。该路径实现了从总线侧到逻辑侧的反馈隔离。

### 4. 逻辑开关与阻抗匹配 (Switch 矩阵)
电路板上设计了一组 DIP 开关，用于配置 RS-485 总线的物理特性：
*   **Switch 1 (偏置电阻控制)**：控制 680Ω 上拉电阻（连接至 `VCC_5V_ISO`）和下拉电阻（连接至 `ISO_GND`）是否接入 A/B 差分线。开启此开关可确保总线在空闲状态下保持确定的电平，防止噪声引发误码。
*   **Switch 2 (模式切换)**：位于信号传输路径中，用于控制内部信号的流向。在特定配置下，决定信号是进入主控进行处理，还是处于直通/旁路状态（具体取决于电路中该开关对 TX/RX 链路的截断或连通逻辑）。
*   **Switch 3 (终端电阻控制)**：控制 120Ω 终端匹配电阻的接入。当设备位于 DMX 总线的末端时，需闭合此开关以消除信号反射，保证长距离通讯的信号完整性。

### 5. 接口物理连接
*   **XLR 接口**：配备标准的 **XLR-5** 和 **XLR-3** 接口（涵盖公头与母头），严格遵循 DMX-512 引脚定义：
    *   Pin 1: `ISO_GND` (信号地)
    *   Pin 2: `Data -` (对应 RS-485 的 B 线)
    *   Pin 3: `Data +` (对应 RS-485 的 A 线)
*   **扩展接口**：提供 **HT3.96** 端子，与 XLR 接口的 A、B、GND 信号并联，便于工业现场直接接线使用。

### 6. M5-Bus 引脚映射
电路通过 M5-Bus 排母与主机连接，关键引脚定义如下：
*   **GPIO 2**: `UART_TX` (数据发送)
*   **GPIO 22**: `UART_RX` (数据接收)
*   **GPIO 21**: `RS485_EN` (收发使能控制)
*   **HPWR**: 9-24V 电源输入/输出总线
*   **5V / 3V3 / GND**: 系统电源与公共地（逻辑侧）

## 补充信息

基于原始文档与之前的分析，对 **M5Stack Base DMX (SKU: M128)** 原理图的补充解析如下：

### 1. 双路 RS-485 链路与主子板拓扑架构
*   **输入/输出物理分离**：电路并非简单的单节点设计，而是采用了 **两路独立的 RS-485 电路结构**，物理上区分了 DMX **输入接口**（通常对应母头）和 **输出接口**（通常对应公头）。这种架构对应了文档中提到的主板（Base DMX）与子板（Base DMX Sub）的分离式设计，子板负责承载物理接口及其特定的阻抗网络。
*   **信号流向逻辑**：设计支持 DMX 协议的菊花链（Daisy Chain）特性，内部信号链路被明确划分为上游输入链路和下游输出链路，而非单一的双向总线。

### 2. Switch 2 的信号级联路由控制
*   **旁路 (Bypass) 与分离模式**：**Switch 2** 是这一架构的核心路由控制器。它是一个内部双掷开关，用于切换 **输入级** 与 **输出级** 之间的连接关系：
    *   **旁路模式（Bypass）**：将输入接口接收到的 RS-485 信号直接物理导通至输出接口。此时设备作为总线上的透明节点或侦听节点，信号在经过设备时不做处理直接向下游传输。
    *   **分离模式（Separation）**：断开输入与输出之间的直接物理连接。输入信号仅进入 M5 主机进行处理，输出接口的信号完全由 M5 主机生成。此模式用于设备作为独立的 DMX 控制器或信号中继器/处理器。

### 3. 端口专属性的阻抗匹配网络
三组 DIP 开关的功能严格对应物理端口的输入/输出属性，而非作用于同一总线节点：
*   **Switch 1 (输出级偏置)**：仅控制 **输出接口 (Output Interface)** 的 680Ω 上拉/下拉偏置电阻。这确保了当设备作为信号源向后级发送信号时，输出线路在空闲状态下的电平稳定性。
*   **Switch 3 (输入级终端)**：仅控制 **输入接口 (Input Interface)** 的 120Ω 终端匹配电阻。这符合 DMX512 标准，用于当该设备位于控制链路的最末端时进行阻抗匹配，吸收信号反射。

### 4. M5-Bus 电源补充
*   **双向供电特性**：虽然设备主要通过 DC-JACK 供电并通过降压电路为自身逻辑供电，但 M5-Bus 连接中的 **HPWR** 引脚（Pin 25, 27, 29）是直通的。这意味着连接在 Base DMX 上的外部 9-24V 电源会直接馈送给 M5 主机，为主机及其电池充电（BAT Pin 30 在 Bus 定义中存在，确认了与主机电源管理的连接性）。
