## 原理图分析

以下是基于 M5Stack Bala2 自平衡机器人硬件架构与技术细节的原理图技术性描述：

### 1. 核心控制器与双核架构
*   **主控单元 (Master Node)**: 采用 ESP32-D0WDQ6-V3 SoC（M5Stack Gray Core 核心板），基于 Xtensa® 32-bit LX6 双核架构，主频 240MHz。负责顶层应用逻辑、PID 平衡算法运算、Wi-Fi/蓝牙无线通讯、TFT 屏幕刷新以及音频输出。
*   **协处理单元 (Slave Node)**: 底座内部集成 STM32F030C8T6 微控制器（ARM Cortex-M0）。作为 I2C 从机运行，主要负责实时性要求高的电机 PWM 生成、编码器脉冲计数、舵机信号发生以及底层运动控制逻辑。
*   **双核通讯链路**: ESP32 与 STM32 之间通过 I2C 总线连接。ESP32 为 I2C 主机，STM32 为 I2C 从机，设备地址设定为 `0x3A`。通讯内容包括 ESP32 下发的电机速度/方向指令、舵机角度指令，以及 STM32 上报的编码器计数与底座状态数据。

### 2. 动力驱动系统电路
*   **直流电机驱动**: 采用 HR8833 双通道 H 桥电机驱动芯片。
    *   **控制信号**: STM32 的 GPIO 输出 PWM 信号至 HR8833 的输入端（AIN1/AIN2, BIN1/BIN2），控制 H 桥的通断与电流方向，从而调节 N20 电机的转速与转向。
    *   **输出级**: HR8833 的输出端（AOUT1/AOUT2, BOUT1/BOUT2）直接连接至左右轮 N20 减速电机。
*   **编码器反馈回路**: N20 电机尾部的霍尔编码器输出 A/B 相正交脉冲信号，直接连接至 STM32 的定时器输入捕获引脚。STM32 通过硬件计数器解算电机实时转速与方向，形成底层的速度闭环控制。
*   **舵机控制接口**: 电路设计了 8 路 PWM 输出通道，全部由 STM32 负责生成控制信号。其中 4 路信号引出至底座外部接口，另外 4 路预留于底座内部接口。舵机电源通常直接取自电池电压或经稳压后的高电流电源轨。

### 3. 传感器与姿态感知网络
*   **惯性测量单元 (IMU)**: M5Stack Gray Core 内部集成 MPU6886（6 轴 MEMS，包含 3 轴陀螺仪与 3 轴加速度计）与 BMM150（3 轴磁力计）。
*   **总线连接**: 传感器挂载于 ESP32 的 I2C 总线上。
*   **数据流向**: ESP32 通过 I2C 高速读取 MPU6886 的加速度与角速度原始数据，结合 BMM150 的磁场数据，内部运行姿态解算算法（如卡尔曼滤波或互补滤波）得出机身倾角，进而输入 PID 控制器计算所需的电机扭矩补偿值。

### 4. 电源管理系统 (PMS)
*   **供电输入**: 系统由内置 1200mAh 锂聚合物电池供电。
*   **充电管理**: 集成电池充电管理电路，输入端为 USB Type-C 接口。当外部电源接入时，充电 IC 对锂电池进行恒流/恒压充电。
*   **电压调节 (LDO/DC-DC)**:
    *   **VBAT**: 电池直连电压，主要供给电机驱动芯片（HR8833）功率级和舵机接口，以确保动力输出。
    *   **3.3V Logic**: 通过 LDO 稳压器将 VBAT 转换为 3.3V，供给 ESP32、STM32、传感器（MPU6886/BMM150）、TF 卡及逻辑电路。
*   **电源路径**: 电池电压经由底座触点或物理连接器传输至 Gray Core 主控板与底座驱动板。

### 5. 外设模块与信号接口
*   **显示输出**: 2 英寸 ILI9342C TFT LCD 控制器通过 SPI 总线与 ESP32 连接。ESP32 发送显示数据与命令控制屏幕刷新。
*   **音频系统**: 包含 I2S 功率放大器芯片（如 NS4168），驱动 1W-0928 扬声器。ESP32 通过 I2S 接口输出数字音频信号。
*   **存储扩展**: MicroSD (TF) 卡槽通过 SPI 总线连接至 ESP32，用于存储固件、日志或媒体资源。
*   **Grove 扩展系统**:
    *   **Port A (I2C)**: 引脚 G22 (SCL), G21 (SDA)，连接至 ESP32 I2C 控制器，支持外部 I2C 传感器扩展。
    *   **Port B (I/O)**: 引脚 G26 (DAC), G36 (ADC)，连接至 ESP32，支持模拟信号输入输出。
    *   **Port C (UART)**: 引脚 G16 (RX), G17 (TX)，连接至 ESP32 的 UART 控制器，用于串行通讯扩展。

### 6. 关键控制回路逻辑总结
1.  **感知**: ESP32 读取 I2C 总线上的 MPU6886 数据，获取当前倾斜角度。
2.  **决策**: ESP32 内部运行 PID 算法，计算维持平衡所需的目标速度和转向，将控制指令打包并通过 I2C 发送给从机 STM32 (地址 0x3A)。
3.  **执行**: STM32 解析 I2C 指令，调整 PWM 占空比输出给 HR8833，驱动 N20 电机动作。
4.  **反馈**: N20 编码器信号反馈回 STM32，STM32 进行底层转速校正并可将里程计数据回传给 ESP32。

### 1. 模块化硬件拓扑与总线物理连接
*   **物理层叠架构 (Stacking Architecture)**: 系统硬件逻辑上分为**计算感知层 (M5Stack Gray)** 与 **驱动执行层 (Bala2 Base)**。两者通过 M5-Bus（底部的 MBUS 插座）进行物理和电气连接。
*   **总线跨板传输**:
    *   **电源轨**: 电池能量从 Base 底座通过 MBUS 的 `BAT` 和 `5V` / `3.3V` 引脚向上传输给 Gray 主控供电。
    *   **通讯轨**: 控制指令通过 MBUS 上的 `SDA` (G21) 和 `SCL` (G22) 跨板传输。
*   **机械兼容性**: 底座外壳及 PCB 设计兼容 LEGO Technic 结构标准，允许通过标准孔位进行机械扩展。

### 2. 传感器电磁兼容性 (EMC) 约束
*   **磁力计干扰规避**: 虽然原理图中包含 BMM150 磁力计（位于 Gray 主控层），但在硬件集成时存在**磁干扰敏感性**。
*   **硬件修正要求**: 由于 M5Stack Gray 核心模块通常在背部内置强磁铁（用于堆叠吸附），在 Bala2 架构中，为保证 BMM150 提供正确的 Yaw（偏航角）数据以辅助 PID 定向，**必须物理移除 Gray 主控内的磁铁**，否则会导致磁力计读数饱和或错误，进而影响闭环控制的稳定性。

### 3. 人机交互 (HMI) 硬件逻辑映射
*   **校准模式信号路径**: 文档提及的校准操作依赖于 M5Stack Gray 的板载按键，其 GPIO 硬件映射如下，直接参与固件的状态机切换：
    *   **Button A (GPIO 39)**: 对应校准参数调整（-）。
    *   **Button B (GPIO 38)**: 对应保存参数/确认，配合电源键（复位）进入 Bootloader 或校准模式。
    *   **Button C (GPIO 37)**: 对应校准参数调整（+）。
*   **GPIO 复用**: 上述按键 GPIO 与扩展接口无冲突，独立于 I2C 总线和电机控制逻辑。

### 4. 扩展接口电气特性补充
*   **Port B (Analog/Digital)**: 明确引脚 G26 定义为 **DAC** (数模转换输出)，G36 定义为 **ADC** (模数转换输入，仅输入)，这意味着 Port B 适合连接模拟传感器或输出模拟波形，而非仅作普通数字 IO。
*   **Port C (UART)**: 明确为硬件串口，G16 (RXD2) / G17 (TXD2)，直接连接 ESP32 的 UART2 控制器，用于与其他串口设备（如激光雷达、GPS）的高速通讯。
