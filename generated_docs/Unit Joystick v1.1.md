### 1. 核心控制器与系统架构
*   **主控芯片**：采用 Atmel **ATmega8A-AU** (8-bit AVR 微控制器) 作为核心处理单元。该芯片在电路中充当 I2C 从机设备（Slave），负责底层硬件的模拟量采集与数字量处理。
*   **功能逻辑**：MCU 内部运行固件，通过硬件 ADC（模数转换器）实时采样摇杆电位器的电压值，并监测按键状态。所有数据被存储在特定寄存器中，等待 I2C 主机（Host）读取。

### 2. 电源管理电路
*   **电源输入**：电源通过 **HY2.0-4P** 接口（Grove 接口）的 Pin 2 (5V) 输入。
*   **稳压与滤波**：

    *   电源路径上配置有去耦电容（通常为 100nF 和 10uF 组合），分别放置在 LDO 输入输出端及 ATmega8A 的电源引脚（VCC/AVCC）附近，用于滤除电源噪声，保证 ADC 采样的稳定性。
    *   GND 网络连接至 HY2.0-4P 的 Pin 1 以及电路板的大面积铺地。

### 3. 输入外设电路（摇杆模块）
*   **模拟输入（X/Y 轴）**：
    *   摇杆内部包含两个 **10kΩ 电位器**，分别对应 X 轴和 Y 轴。
    *   电位器的两端分别连接 VCC (3.3V) 和 GND，形成分压电路。
    *   电位器的滑动端（Wiper）分别连接至 ATmega8A 的 **ADC 输入引脚**（通常为 PC0/ADC0 和 PC1/ADC1）。
    *   MCU 将采集到的模拟电压转换为 0-255 的 8 位数字值（uint8_t），分别映射摇杆在 X 和 Y 方向的位置。
*   **数字输入（Z 轴/按键）**：
    *   摇杆集成的下压按键一端连接 GND，另一端连接至 ATmega8A 的 **GPIO 引脚**（如 PD2 或 PB1）。
    *   该 GPIO 引脚在 MCU 内部或外部配置了 **上拉电阻**。
    *   **逻辑状态**：未按下时，引脚被拉高（逻辑 1）；按下时，引脚短路至地（逻辑 0）。MCU 检测该电平变化以判断按键状态。

### 4. 通讯接口与信号路径
*   **物理接口**：使用 HY2.0-4P 连接器连接外部主控。
    *   Pin 3: **SDA** (I2C 数据线)。
    *   Pin 4: **SCL** (I2C 时钟线)。
*   **I2C 总线连接**：
    *   SDA 和 SCL 信号线直接连接至 ATmega8A 的对应硬件 I2C 引脚（PC4/SDA, PC5/SCL）。
    *   总线上通常配置有 **上拉电阻**（如 4.7kΩ 或 10kΩ），将 SDA 和 SCL 拉至 VCC，以确保总线空闲时处于高电平状态。
*   **通讯协议**：
    *   ATmega8A 设定为 I2C 从机模式，默认设备地址为 **0x52**。
    *   通讯逻辑支持单字节读取或连续读取，寄存器映射通常为：X 轴数据、Y 轴数据、按键状态。

### 5. 辅助与编程电路
*   **ISP 编程接口**：原理图或 PCB 上预留有 AVR ISP (In-System Programming) 接口测试点，包含 MOSI, MISO, SCK, RESET, VCC, GND 信号，用于出厂时烧录 ATmega8A 的固件。
*   **复位电路**：ATmega8A 的 RESET 引脚通常通过一个 10kΩ 电阻上拉至 VCC，防止芯片意外复位。
