## 原理图分析

**核心控制器与系统架构**
该原理图设计的核心是基于 **ESP32-PICO** 系统级封装（SiP）芯片的 I2C 转 SPI 桥接架构。ESP32-PICO 作为主控单元，被配置为 I2C 从机设备（默认地址 **0x3E**），负责接收来自主机的指令和数据，并管理内部帧缓冲区。

**显示驱动子系统**
显示输出由 **ST7789V2** LCD 驱动芯片管理，用于驱动分辨率为 **135 x 240** 的 1.14 英寸彩色 LCD 面板。ESP32-PICO 与 ST7789V2 之间采用高速 **SPI 接口** 通讯，并启用 DMA 以提升刷新性能。两者之间的连接关系包括：
*   **MOSI:** 主出从入数据线。
*   **SCLK:** 串行时钟线。
*   **CS:** 片选信号。
*   **DC:** 数据/命令选择控制线。
*   **RST:** 硬件复位信号。
此外，电路包含背光控制电路，由 ESP32 通过 PWM 信号驱动，支持 0-255 级的亮度调节。ST7789V2 芯片周围配置了去耦电容以确保高频工作的稳定性。

**电源管理电路**
系统电源通过 **HY2.0-4P (Grove Port A)** 接口接入，输入电压为 **5V**。该 5V 输入经过板载稳压电路降压至 **3.3V**，用于为 ESP32-PICO 控制器、LCD 逻辑电路及 SPI 总线供电。电源网络设计包含了必要的滤波处理。

**通讯与外设接口**
外部连接通过 Grove 接口实现，引脚定义包括 **SDA、SCL**（I2C 信号线）以及 **5V、GND**（电源线）。该接口是 Unit LCD 与外部主机通讯的主要物理通道。

**逻辑控制与固件维护**
为了便于固件烧录和调试，原理图上引出了 **UART 编程接口**，具体包括：
*   **TXD / RXD:** 串行通讯发送与接收引脚。
*   **IO0:** 启动模式选择引脚（用于进入下载模式）。
*   **EN/RST:** 芯片复位与使能引脚。
这组接口允许使用外部编程器（如 ESP32-Downloader）进行直接刷写。此外，电路逻辑设计也支持通过 I2C 总线协议进行固件更新。

## 补充信息

**显示面板技术规格补充**
*   **色彩深度与类型:** 采用 IPS 面板技术，提供全视角显示能力；支持 RGB666 色彩格式，可显示 262,144 种颜色。
*   **像素物理特性:** 像素间距为 0.1101 (H) x 0.1038 (V) mm。

**I2C 通讯协议与逻辑控制详解**
*   **总线速率:** 支持最高 400kHz 的 I2C 通信速率。
*   **动态地址管理:** 支持通过软件指令（`CHANGE_ADDR`）修改默认的 7 位 I2C 地址（0x3E），具备位反转校验机制。
*   **缓冲区与流控:** ESP32 内部实现了命令缓冲区机制，I2C 通讯单元与绘图处理单元并行运行。主机需通过 `READ_BUFCOUNT` 指令监控缓冲区剩余空间（0-255），以防止在大数据量传输（如全屏填充）时发生缓冲区溢出。
*   **异常处理:** 若在命令传输序列中发生 I2C STOP 或 RESTART 信号，未完成的命令序列将被丢弃而不执行。

**图形数据处理与固件特性**
*   **数据编码支持:** 固件原生支持 RLE (Run-Length Encoding) 运行长度编码压缩格式，可针对 RGB565 和 RGB888 数据进行解压显示，以降低 I2C 带宽占用。
*   **色彩格式兼容:** 支持多种写入格式，包括 RGB332 (8-bit)、RGB565 (16-bit)、RGB888 (24-bit) 以及带 Alpha 通道的 ARGB8888 (32-bit)。
*   **硬件合成能力:** 支持基于 Alpha 通道的图像透明合成操作，允许新绘制内容与现有帧缓冲区内容进行混合。

**物理结构特性**
*   **固定方式:** 屏幕背部集成磁吸结构设计，支持吸附于金属表面。
*   **材质:** 外壳采用 PC（聚碳酸酯）材质。
