*   **整流与隔离**：输入端包含桥式整流电路，确保无论网线线序极性如何均能正常取电。

    *   **二级稳压**：5V 电源经由 LDO（低压差线性稳压器）转换为 **3.3V**，为 ESP32、W5500、摄像头及逻辑电路供电。
*   **供电特性**：设计仅通过 RJ45 接口供电，电路中未设计电池充放电管理单元。

### 3. 以太网通讯电路
*   **以太网控制器**：采用 **W5500** 硬件 TCP/IP 协议栈芯片，处理以太网底层数据包。
*   **SPI 通讯接口**：W5500 作为从机通过 SPI 总线与主机 ESP32 通讯：

### 4. 图像采集系统 (OV2640)
*   **传感器接口**：OV2640 摄像头通过 **DVP (Digital Video Port)** 接口与 ESP32 直接相连。
*   **数据总线**：8 位并行数据线连接如下：

    *   **PCLK (像素时钟)**：连接至 **GPIO21**。
    *   **VSYNC (场同步)**：连接至 **GPIO22**。
    *   **HREF (行同步)**：连接至 **GPIO26**。
*   **系统时钟**：
    *   **XCLK**：由 ESP32 的 **GPIO27** 输出时钟信号驱动摄像头工作。
*   **配置与控制**：
    *   **SCCB 总线**：利用 I2C 协议进行寄存器配置，SIOC (SCL) 连接 **GPIO12**，SIOD (SDA) 连接 **GPIO14**。
    *   **复位控制**：**RESET** 引脚连接至 **GPIO15**，用于硬件复位摄像头模块。
    *   **供电**：包含专用的 LDO 电路为摄像头提供模拟（2.8V）和数字核心（1.2V）电压。

*   **UART / 程序烧录接口**：

### 6. 整体电路逻辑总结
系统通过 RJ45 接口同时接收电源和数据。PoE 电路分离出高压直流电并降压供给全板；W5500 负责将以太网信号转换为 SPI 信号传送给 ESP32。ESP32 利用 DVP 接口高速读取 OV2640 的图像数据，存入 PSRAM，并通过 Wi-Fi 或 W5500 以太网接口将视频流传输出去。GPIO0 和 GPIO15 等关键引脚在设计上兼顾了外设控制（LED、Camera Reset）与 ESP32 的启动时序配置要求。
