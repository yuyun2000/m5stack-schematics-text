## 原理图分析

以下是基于 M5Stack Base LAN PoE (SKU: K012-C) 硬件规格与原理图的详细技术解析：

### 1. 核心以太网控制器 (W5500)
*   **控制器型号**：采用 WIZnet W5500 全硬件 TCP/IP 嵌入式以太网控制器，集成了 TCP/IP 协议栈、10/100M 以太网数据链路层（MAC）及物理层（PHY）。
*   **时钟电路**：芯片外接 25MHz 无源晶振，为内部 PLL 提供基准时钟。
*   **主控通讯接口 (SPI)**：
    *   **协议**：W5500 作为 SPI 从机通过 M5-Bus 与主控（ESP32）进行通讯。
    *   **SCK (时钟)**：连接至 M5-Bus 的 **G18** 引脚。
    *   **MISO (主入从出)**：连接至 M5-Bus 的 **G19** 引脚。
    *   **MOSI (主出从入)**：连接至 M5-Bus 的 **G23** 引脚。
    *   **SCSn (片选)**：连接至 M5-Bus 的 **G26** 引脚，低电平有效。
*   **控制信号**：
    *   **RSTn (复位)**：连接至 M5-Bus 的 **G13** 引脚，用于硬件复位 W5500。
    *   **INTn (中断)**：连接至 M5-Bus 的 **G34** 引脚，当以太网事件（如接收数据包）发生时，拉低该引脚通知主控。
*   **物理层连接**：W5500 的 TXP/TXN 和 RXP/RXN 差分信号线连接至 RJ45 插座内部的隔离变压器。

### 2. 电源与 PoE 电路
*   **PoE 供电输入**：
    *   电路符合 IEEE802.3 af 标准。
    *   通过 RJ45 接口的空闲线对提取电力。根据原理图定义，**J4/J5** 引脚汇流作为 **VC-**，**J7/J8** 引脚汇流作为 **VC+**。
    *   VC+ 与 VC- 线路输入至板载 PoE 模块（隔离型 DC-DC 转换器），将以太网高压转换为系统所需的低压直流电（通常为 5V）。
*   **外部直流输入**：
    *   包含一路 DC 电源输入电路，支持 **9V~24V** 宽电压输入。
    *   输入端通常包含防反接二极管保护。
*   **电源汇流与稳压**：
    *   PoE 模块输出电压与外部 DC 输入电压通过二极管或电源管理逻辑汇流。
    *   汇流后的电源经过降压电路（Buck Converter）稳压至 5V，连接至 M5-Bus 的 **5V** 和 **HPWR** 网络，为堆叠的主机及底座逻辑电路供电。

### 3. 串口转换子系统
*   **RS485 通讯电路**：
    *   **收发器**：采用 **SP3485EE** 芯片，这是一款 3.3V 供电的半双工 RS-485 收发器。
    *   **信号路径**：芯片的 A、B 差分信号引脚连接至 HT3.96 端子。
    *   **控制逻辑**：RO (接收输出) 和 DI (驱动输入) 连接至 TTL 侧逻辑；RE (接收使能) 和 DE (驱动使能) 通常由主控 GPIO 或硬件流控逻辑控制。
*   **RS232 通讯电路**：
    *   **收发器**：采用 **MAX232**（或同系列电平转换芯片）。
    *   **电荷泵**：芯片周围配置 4 个电解电容或陶瓷电容，用于内部电荷泵升压/负压产生，以符合 RS-232 双极性电平标准（±10V左右）。
    *   **信号路径**：T1OUT/R1IN 连接至 HT3.96 端子的 TX/RX 引脚。
*   **拨码开关 (Dip Switch) 路由逻辑**：
    *   板载设有多位拨码开关，用于物理切换 UART 信号的流向。
    *   通过改变开关状态，可以将 M5-Bus 上的特定 GPIO（通常为 UART 引脚，如 G16/G17）选择性地导通至 SP3485EE (RS485) 或 MAX232 (RS232) 的 TTL 侧。
    *   该设计允许 RS485 和 RS232 共用主控的一组 UART 控制器，但在同一时刻只能激活其中一种物理接口通讯。

### 4. 接口与连接器
*   **RJ45 接口**：
    *   内置网络变压器（Magnetics），实现信号耦合与电气隔离。
    *   引出中心抽头用于 PoE 取电。
    *   集成 Link（连接）和 Act（活动）状态指示灯 LED。
*   **HT3.96 端子 (6-Pin)**：
    *   作为外部工业控制接口。
    *   **引脚定义**：包含 RS485-A、RS485-B、RS232-TX、RS232-RX、以及电源输出（VCC）和地（GND）。
*   **M5-Bus (30-Pin)**：
    *   作为与 M5Stack 主机堆叠的连接桥梁。
    *   **电源引脚**：5V, 3.3V, BAT, GND。
    *   **信号引脚**：SPI 总线 (G18, G19, G23)、控制信号 (G13, G26, G34) 以及通过拨码开关配置的 UART 信号引脚。

### 5. 逻辑与信号流
*   **以太网数据流**：数据包由 ESP32 处理 -> 通过 SPI 总线 (G18/19/23/26) 传输至 W5500 -> W5500 进行协议封装 -> 差分信号经 RJ45 变压器耦合 -> 网线传输。
*   **串口信号流与隔离**：
    *   由拨码开关决定物理路径：
        *   **路径 A (RS485)**：M5-Bus TTL 信号 -> 拨码开关闭合 RS485 侧 -> SP3485EE -> 差分电平 (A/B) -> HT3.96 接口。
        *   **路径 B (RS232)**：M5-Bus TTL 信号 -> 拨码开关闭合 RS232 侧 -> MAX232 -> RS232 电平 -> HT3.96 接口。
    *   这种设计在物理层面上隔离了未使用的串口模块，防止信号干扰。
*   **电源流**：高压输入 (PoE 48V 或 DC 12V/24V) -> 电源管理模块 -> 5V 系统电压 -> M5-Bus -> 供电给主机 ESP32 及板载 W5500/SP3485/MAX232 芯片。
