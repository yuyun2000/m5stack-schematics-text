## 原理图分析

以下是基于 M5StickC-Plus 硬件设计的原理图技术解析与结构化描述：

### 1. 核心处理器与射频电路
*   **主控芯片**：采用 **ESP32-PICO-D4** SiP（系统级封装）模组。该模组内部集成了 ESP32 双核 Wi-Fi/蓝牙组合芯片、4MB SPI Flash 存储器、晶振及必要的滤波电容。
*   **射频天线**：模组通过射频引脚连接至板载 **3D 天线**，用于 2.4GHz Wi-Fi 和蓝牙通讯，电路包含匹配网络以优化天线性能。

### 2. 电源管理系统 (PMU)
系统电源由 **AXP192** 芯片统一管理，负责电源路径选择、电压转换及电池充电。
*   **输入电源**：
    *   **VBUS_USB**：来自 USB Type-C 接口的 5V 输入，经过保护电路后接入 AXP192。
    *   **VBAT**：连接内置 120mAh 锂电池，AXP192 负责电池的充电管理与放电路径。
*   **电源轨分配**：
    *   **DC-DC1**：输出 3.3V 主系统电压，为 ESP32-PICO-D4、MPU6886 传感器及逻辑电路供电。
    *   **LDO1**：输出持续电源给 **BM8563** 实时时钟芯片，确保关机状态下时间保持。
    *   **LDO2**：为 TFT 屏幕的背光电路供电，可软件调节亮度。
    *   **LDO3**：为 TFT 屏幕的逻辑驱动部分（ST7789v2）供电。
    *   **LDOio0**：专为 **SPM1423** 麦克风供电，实现音频模块的独立电源控制。
    *   **IPSOUT**：系统主 5V 输出，直接连接至 HY2.0-4P (Grove) 接口的 VCC 引脚，为外部传感器供电。
*   **电源控制**：AXP192 的 PWRON 引脚连接至侧边电源按键，支持长按开关机及短按复位功能。

### 3. 显示系统
*   **LCD 屏幕**：采用 1.14 英寸彩色 TFT 屏幕。
*   **驱动控制器**：内置 **ST7789v2** 驱动 IC。
*   **信号接口**：通过 ESP32 的 SPI 接口驱动。
    *   **MOSI (SDA)**: GPIO15
    *   **SCLK (SCK)**: GPIO13
    *   **DC (Data/Command)**: GPIO23
    *   **RST (Reset)**: GPIO18
    *   **CS (Chip Select)**: GPIO5
*   **电源连接**：屏幕背光与逻辑电源分别由 AXP192 的 LDO2 和 LDO3 独立控制。

### 4. 传感器与时钟总线 (I2C)
板载传感器挂载于 ESP32 的 I2C 总线上，总线引脚为 SDA (GPIO21) 和 SCL (GPIO22)。
*   **IMU 传感器**：**MPU6886** 六轴姿态传感器（加速度计 + 陀螺仪），用于运动检测与姿态解算。
*   **RTC 时钟**：**BM8563** 实时时钟芯片，负责提供精确的时间信息，具有独立的定时报警中断输出连接至 ESP32。

### 5. 音频输入输出系统
*   **数字麦克风**：采用 **SPM1423** PDM（脉冲密度调制）数字麦克风。
    *   **CLK**: 连接至 GPIO0。
    *   **DATA**: 连接至 GPIO34（输入专用引脚）。
    *   **电源**: 由 AXP192 的 LDOio0 供电，不使用时可关闭以省电。
*   **蜂鸣器**：板载无源蜂鸣器，信号脚连接至 ESP32 的 GPIO2，通过 PWM 信号驱动发声。

### 6. 用户交互与外设控制
*   **按键**：
    *   **Button A**：连接至 GPIO37（M5 大按键）。
    *   **Button B**：连接至 GPIO39（侧边按键）。
    *   *注：GPIO37 和 GPIO39 为 ESP32 的仅输入引脚 (GPI)，电路中通常包含上拉电阻或利用内部上拉。*
*   **指示灯**：红色 LED 连接至 GPIO10，低电平点亮（或根据具体电路极性设计）。
*   **红外发射**：红外发射二极管连接至 GPIO9，用于红外遥控信号发送。

### 7. 外部接口与扩展
*   **USB Type-C**：用于设备供电（VBUS）及数据通讯。内部通过 USB-TTL 芯片（如 CP2104 或 CH9102）连接 ESP32 的 UART0 (TXD: GPIO1, RXD: GPIO3) 进行程序下载与调试。
*   **Grove 接口 (HY2.0-4P)**：提供 I2C 或通用 I/O 扩展。
    *   引脚定义：GND, 5V (IPSOUT), GPIO33, GPIO32。
*   **HAT 扩展插槽**：顶部 8 Pin 排母，引出了一组 GPIO 及电源（3.3V, 5V, GND），用于堆叠扩展模块。涉及引脚包括 GPIO0, GPIO26, GPIO36, GPIO25 等。

### 8. 特殊逻辑与信号路径
*   **GPIO36/G25 逻辑**：
    *   GPIO36 (ADC1_CH0) 为仅输入引脚，常用于高精度 ADC 采样。在扩展电路设计中，需注意其不可作为推挽输出使用。
    *   GPIO25 为通用 I/O 且具有 DAC 功能。
    *   这两组引脚在连接 HAT 设备时需注意引脚复用的冲突管理。
*   **GPIO0 复用**：
    *   GPIO0 同时作为 PDM 麦克风的时钟线 (CLK) 和 ESP32 的启动模式选择引脚（Boot Pin）。
    *   在电路设计上需确保麦克风电路不会在启动瞬间将 GPIO0 拉低，以免误进入下载模式。
*   **电源保护**：VBUS 输入端通常包含二极管或 MOS 管防倒灌电路，确保在使用 USB 供电时不会向电池反向过充（由 AXP192 内部逻辑管理），同时防止电池电压倒灌回 USB 端口。
