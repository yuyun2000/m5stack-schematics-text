## 原理图分析

以下是基于 M5Stack Atom Display-Lite (SKU:K115-B) 硬件规格与逻辑结构的原理图技术描述：

### 1. 核心控制单元 (MCU Module)
*   **主控芯片**：采用 ESP32-PICO-D4 系统级封装 (SiP) 模组，内部集成 ESP32 双核处理器、4MB SPI Flash 以及 520KB SRAM，负责系统整体逻辑控制、图形数据生成及外设驱动。
*   **连接方式**：Atom-Lite 模块通过底部的 M5-Bus 排母与 Display-Lite 扩展基板进行物理和电气连接。
*   **主要功能**：
    *   作为 SPI 主机 (Master)，向 FPGA 发送图像数据。
    *   作为 I2C 主机，配置视频转换芯片 LT8618SX 的寄存器。
    *   处理用户输入（按键）及状态指示（LED）。

### 2. FPGA 驱动电路 (Gowin GW1NR-9C)
*   **芯片选型**：Gowin (高云) GW1NR-9C FPGA。
*   **输入接口 (SPI)**：
    *   FPGA 作为 SPI 从机 (Slave) 接收来自 ESP32 的高速信号。
    *   **信号映射**：
        *   **CS (片选)**：连接至 M5-Bus 定义的片选引脚，用于激活 FPGA 通信。
        *   **CLK (时钟)**：接收 ESP32 输出的 SPI 时钟信号，用于同步数据传输。
        *   **MOSI (数据输入)**：接收 ESP32 发送的模拟 TFT-LCD 像素数据流。
        *   **MISO (数据输出)**：用于 FPGA 向 ESP32 反馈状态（如有配置）。
    *   M5-Bus 上的物理引脚（2, 4, 6, 8）承载上述 SPI 关键信号。
*   **逻辑功能**：内部逻辑配置为视频时序发生器（Video Timing Generator）。它解析接收到的 SPI 串行数据，并将其重构为并行视频信号。
*   **输出接口**：输出标准的 24-bit RGB 并行数据（R[0-7], G[0-7], B[0-7]）以及视频同步信号（HSYNC, VSYNC, DE, PCLK），直接驱动后级视频转换芯片。

### 3. 视频信号转换电路 (LT8618SX)
*   **芯片选型**：Lontium (龙迅) LT8618SX，是一款低功耗 RGB 转高清数字多媒体信号（HDMI-compatible）的发射芯片。
*   **数据输入**：连接至 FPGA 输出的 24-bit RGB 总线及控制信号线，接收 TTL 电平的并行视频数据。
*   **控制接口 (I2C)**：
    *   **SDA (数据线)**：连接至 ESP32 的 **G25** 引脚 (LT_CSDA)。
    *   **SCL (时钟线)**：连接至 ESP32 的 **G21** 引脚 (LT_CSCL)。
    *   ESP32 通过该 I2C 总线初始化 LT8618SX，配置输出分辨率、色彩空间及音频参数。
*   **信号输出**：将并行 RGB 信号编码为 TMDS 差分信号，通过高清多媒体接口输出至显示设备。

### 4. 电源管理与分配
*   **供电输入**：系统主电源来自 Atom-Lite 的 USB 接口 (5V) 或外部 5V (VIN) 供电。
*   **电压转换与分配**：
    *   **5V 电源轨**：直接为 Atom-Lite 模块供电，并输入至板载 LDO（低压差线性稳压器）。
    *   **3.3V 电源轨**：LDO 输出 3.3V，为 ESP32-PICO-D4 的 IO 接口、LT8618SX 的 I/O 供电以及外设接口提供电源。
    *   **FPGA 专用电源**：通常需要额外的低电压（如 1.2V 核心电压），由基板上的专用电源芯片从 5V 或 3.3V 降压得到。
*   **接地设计**：所有模块（ESP32, FPGA, LT8618SX, 接口）共用 GND 层，确保高速信号传输的参考电平一致性，减少干扰。

### 5. 外设与接口逻辑
*   **人机交互**：
    *   **可编程按键**：物理按键连接至 ESP32 的 **G39** 引脚。电路设计通常包含上拉电阻，按键按下时引脚电平被拉低（Active Low）。
    *   **状态指示灯**：板载一颗 SK6812 (RGB LED)，其数据输入引脚 (DIN) 连接至 ESP32 的 **G27**。该 LED 采用单线串行通讯协议控制颜色与亮度。
*   **扩展接口 (Grove)**：
    *   提供标准的 Grove 接口（HY2.0-4P），引出 5V, GND 以及两路 GPIO/I2C 信号线，用于连接 M5Stack 体系的传感器或执行器。
*   **M5总线 (M5-Bus)**：
    *   作为 Atom-Lite 与扩展基板的连接桥梁，不仅传输电源，还承载关键的 SPI (Pins 2, 4, 6, 8) 和 I2C 信号，确保模块化设计的互联互通。

### 6. 信号流向与系统逻辑
该系统的硬件架构旨在通过低成本 MCU 实现高清视频输出，其完整信号链路如下：

1.  **数据源生成**：ESP32 (Atom-Lite) 在内存中生成图像帧缓冲区数据。
2.  **高速传输**：ESP32 通过 SPI 接口以极高波特率将像素数据发送至 Gowin GW1NR-9C FPGA。
3.  **协议转换 (时序重建)**：FPGA 充当帧缓存与时序控制器，将串行的 SPI 数据流解包，并根据预设的视频时序（如 1280x720 @ 60Hz），实时生成像素时钟 (PCLK) 和 RGB888 并行色彩数据。
4.  **物理层编码**：LT8618SX 接收 FPGA 输出的 RGB 并行信号，将其调制为高速 TMDS 差分信号。
5.  **终端输出**：最终信号通过标准高清多媒体接口输出至显示器，实现最大 720P 分辨率的图像显示。

该硬件基础利用 FPGA 的并行处理能力解决了 ESP32 直接驱动高清屏 IO 速度不足和引脚不够的问题，同时利用 LT8618SX 完成了物理接口的标准化转换。
