*   **时钟电路**：MEGA328 配置有外部晶振电路，为 MCU 提供精确的工作时钟源，确保按键扫描和 I2C 通信的时序稳定性。
*   **复位与去耦**：芯片配置了标准的复位电路，并在电源引脚附近布置了去耦电容，以滤除电源噪声，保证 MCU 在按键操作引起电流波动时的稳定性。

### 2. Faces Panel Bus 接口与电源
*   **接口定义**：电路板通过一个 2x11 (22-pin) 的 Faces Panel Bus 排母与 M5Core 主机连接。
*   **电源网络**：

    *   **GND**：通过 Bus 接口的 Pin 1 和 Pin 2 连接系统共地。
*   **信号直连**：Bus 接口中的 **RXD** (Pin 12) 和 **TXD** (Pin 14) 进行了物理连通，保留了 UART 通路的完整性，虽然本模块主要使用 I2C，但该设计允许串口信号在堆叠系统中透传。

### 3. I2C 通信与逻辑控制
*   **通信协议**：MEGA328 与 M5Core 主机之间通过 I2C 总线进行通信。
*   **设备地址**：固件预设的 I2C 从机地址为 **0x08**。
*   **信号路径**：
    *   **SDA (数据线)**：连接至 Bus 接口的 Pin 16。
    *   **SCL (时钟线)**：连接至 Bus 接口的 Pin 18。
    *   总线上通常配置有上拉电阻，以维持空闲时的高电平状态。
*   **中断机制**：为了提高响应效率，MEGA328 控制一个 **INT** 信号引脚，连接至 Bus 接口的 **Pin 22**。当检测到按键状态变化时，MCU 拉动该引脚触发主机中断，提示主机读取键值，避免了低效的轮询方式。

### 4. 按键输入矩阵
*   **按键布局**：板载 8 个物理机械按键，分别对应功能：**Up (上)、Down (下)、Left (左)、Right (右)、A、B、Start (开始)、Select (选择)**。
*   **电路逻辑**：
    *   每个按键的一端连接至 ATmega328 的特定 GPIO 引脚，另一端连接至 GND。
    *   **触发方式**：采用低电平触发逻辑。GPIO 引脚配置为输入模式（通常启用内部上拉或外部上拉电阻）。当按键被按下时，线路闭合导通至地，MCU 检测到引脚电平由高变低，从而识别按键动作。

### 5. 程序烧录接口 (ISP)
*   **接口功能**：板载预留了标准的 6-pin ISP (In-System Programming) 下载接口，用于 ATmega328 的固件烧录、Bootloader 更新或二次开发。
*   **管脚定义**：接口包含了 **MOSI、MISO、SCK** (SPI 通信线)，以及 **RESET** (复位)、**VCC** (电源)、**GND** (地) 引脚。

### 6. 信号流向总结
1.  用户按下 Gamepad 面板上的任意按键（如 Button A）。
2.  对应的 ATmega328 GPIO 引脚电平被拉低。
3.  ATmega328 内部程序通过扫描捕获该电平变化，更新内部寄存器中的键值状态。
4.  ATmega328 通过 Pin 22 (INT) 发送中断信号通知 M5Core 主机。
5.  主机响应中断，通过 I2C 总线 (Pin 16 SDA, Pin 18 SCL) 以地址 0x08 读取 ATmega328 的寄存器数据，获取当前按键值。
