*   **SPI 通信接口**：W5500 作为 SPI 从机设备连接至 M5-Bus。
    *   **SCLK (时钟)**：连接至 M5-Bus 的 **GPIO18**。
    *   **MISO (主机输入从机输出)**：连接至 M5-Bus 的 **GPIO19**。
    *   **MOSI (主机输出从机输入)**：连接至 M5-Bus 的 **GPIO23**。
    *   **SCSn (片选)**：连接至 M5-Bus 的 **GPIO26**。该引脚低电平有效，用于激活 W5500 的 SPI 通讯。
*   **控制信号**：
    *   **RSTn (复位)**：连接至 **GPIO13**。低电平脉冲用于硬件复位 W5500 内部寄存器。
    *   **INTn (中断)**：连接至 **GPIO34**。配置为开漏输出或推挽输出（取决于外围上拉电阻配置），当 W5500 接收到数据或发生网络事件时，拉低该引脚通知主控 MCU。
*   **时钟电路**：芯片外接 25MHz 无源晶振（XTL），配合负载电容为内部 PLL 提供基准时钟。
*   **电源滤波**：W5500 的各电源引脚（VDD, AVDD）旁路有去耦电容（0.1uF/10uF），以滤除高频噪声。

### 2. 电源管理电路 (Power Section)

电源部分采用降压（Buck）拓扑结构，以支持宽电压输入并为系统提供稳定的工作电压。

*   **输入级**：通过 **HT3.96-4P** 端子接入外部电源，支持 **9V-24V** 直流输入。输入端包含防反接二极管保护电路。
*   **DC-DC 降压电路**：主稳压器采用开关电源芯片（如 XL1509-5.0 或类似 Buck 转换器），将输入的 9-24V 高压转换为系统主电源 **5V**。该 5V 电源不仅供给 Base 底板，还通过 M5-Bus 的 5V 引脚向上层堆叠的主控 Core 供电。
*   **LDO 稳压电路**：5V 电源轨进一步通过低压差线性稳压器（LDO，如 AMS1117-3.3）转换为 **3.3V**。该电压专门用于为 W5500 芯片、RJ45 接口的中心抽头偏置以及逻辑电平电路供电。
*   **保护机制**：输入端并联有电解电容用于储能和滤波，防止输入电压浪涌；DC-DC 输出端包含电感和续流二极管，构成完整的降压回路。

### 3. 网络接口 (RJ45 Interface)

*   **差分信号连接**：W5500 的 TXP/TXN（发送差分对）和 RXP/RXN（接收差分对）直接连接至 RJ45 内部变压器的初级侧。
*   **偏置与滤波**：RJ45 变压器中心抽头连接至 3.3V 电源，并通过滤波电容接地，以抑制共模噪声。
*   **状态指示灯**：
    *   **LINK LED**：连接至 W5500 的 LINKLED 引脚，指示物理层连接状态。
    *   **ACT LED**：连接至 W5500 的 ACTLED 引脚，指示数据传输活动。

### 4. 扩展通信模块 (RS485/RS232 Expansion)

该部分设计为模块化子板焊盘，允许用户选择焊接 **TTL-to-RS485** 或 **TTL-to-RS232** 转换模块。

*   **UART 链路**：扩展模块的 TTL 侧数据线连接至主控的 UART 接口。

    *   方向控制（DE/RE）通常通过硬件自动流控电路实现（利用 TX 信号驱动三极管控制），或直接拉高/拉低以适配特定工作模式。
*   **RS232 模式**：
    *   采用 RS232 驱动芯片（如 MAX232）。
    *   利用电荷泵电容产生 RS232 标准所需的正负电压。
    *   输出端为 TX/RX 电平信号，连接至 HT3.96 端子。

### 5. M5-Bus 与逻辑切换电路

*   **M5-Bus 接口**：标准的 2x15P 排母连接器，通过它与上层 M5Stack Core 进行电气连接，传递 GPIO、I2C、SPI、UART 及电源信号。
*   **逻辑切换开关 (Switch Circuit)**：
    *   原理图中包含一组滑动开关或拨码开关（Switch）。
    *   **功能**：该开关串联在扩展通信模块（RS485/232）的 TTL 数据线与 M5-Bus 的 GPIO 引脚之间。
    *   **路由逻辑**：
        *   **RX 路径**：开关控制 RS485/232 的 RX 输出是否导通至 **GPIO5**。
        *   **TX 路径**：开关控制 RS485/232 的 TX 输入是否导通至 **GPIO15**。
    *   **设计目的**：由于 GPIO5 和 GPIO15 在部分 M5Stack 主控（如 ESP32）上可能被复用为其他功能（如 LCD 控制或片选），该物理开关允许用户在不移除模块的情况下断开 UART 连接，从而避免总线冲突或引脚干扰。
