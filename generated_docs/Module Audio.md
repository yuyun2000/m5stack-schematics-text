*   **模拟开关与标准切换 (CTIA/OMTP)**：
    *   **芯片**：采用 **FSUSB42UMX**（或同等功能的 DPDT 模拟开关）。

    *   **控制逻辑**：开关的 `Sel` 引脚由 STM32 的 **`HP_MODE_SET`** (PA1) 引脚驱动。
        *   当 STM32 输出相应电平时，FSUSB42UMX 交叉切换 MIC 与 GND 的连接路径，从而在硬件层面兼容不同标准的耳机，无需转接头。
*   **插拔检测**：
    *   TRRS 插座包含机械开关引脚，该引脚连接至 STM32 的 **`HP_DET`** (PA4) 输入端。当耳机插入时，引脚电平变化，STM32 检测到中断后可通过 I2C 寄存器状态通知主机。

*   **总线布局**：ES8388 的 I2S 信号线（MCLK, BCLK, LRCK, SDIN, SDOUT）并不直接硬连线固定在 M5-Bus 的特定 GPIO 上。

### 4. 电源与辅助电路

*   **电源分配**：
    *   **5V 电源**：来自 M5-Bus 的 VBUS，主要用于为 RGB LED 供电。
    *   **3.3V 电源**：来自 M5-Bus 的 3V3，为 STM32 MCU、ES8388 的数字/模拟部分（AVDD/DVDD）及模拟开关供电。电源输入端均配置有去耦电容以滤除噪声。
*   **RGB LED 指示灯**：
    *   板载 3 颗 **WS2812C-2020** 串行 RGB LED。
    *   **驱动逻辑**：STM32 的 **`LED_DAT`** (PA7) 引脚连接第一颗 LED 的 DIN，后续 LED 级联。MCU 负责生成符合 WS2812 时序的控制信号。

### 5. 通讯总线

*   **I2C 总线级联**：
    *   M5-Bus 的 SDA 和 SCL 信号线同时连接到：
        1.  STM32G030 (控制逻辑/切换/灯光)。
        2.  ES8388 (音频配置)。
    *   主机（Master）通过同一条 I2C 总线，利用不同的从机地址（0x33 和 0x10）分别对逻辑控制部分和音频编解码部分进行操作。
